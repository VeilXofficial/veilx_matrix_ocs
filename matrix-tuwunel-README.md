# Matrix 私有通讯服务器 · 一键部署（tuwunel 轻量版）

**你的服务器，你的数据 —— 为商业机密保护而生的自托管团队通讯系统。**

本版基于 **tuwunel**（Rust 引擎，内置数据库、**免 PostgreSQL**）：更省资源、更稳、**能像 Telegram 一样发大文件/大图/长视频**，2GB 内存即可撑起中型团队。默认端到端加密、默认元数据最小化、默认邀请制。一行命令部署，不懂网络技术也能用 —— 每个选项都有大白话说明。

推荐使用 **Element X** 客户端；**VeilX** 客户端正在开发中（比 Element X 更美观易用、功能更多、修 Bug 更快、代码开源可审计，运维团队设立在英国、新加坡、日本等地区）。客户资料、报价合同、内部讨论、语音视频会议 —— 全部只存在于你自己的服务器上。基于 [Matrix](https://matrix.org) 开放协议，源码公开可审计（非商业免费）。

## 📋 开始前的准备清单（3 样）

| 需要什么 | 具体要求 | 去哪儿弄 |
|---|---|---|
| **一台海外云服务器** | Ubuntu 22.04 / 24.04（或 Debian 11+），有公网 IP。**按人数选内存**（见下）。**大文件用户请预留足够磁盘** | 推荐 **CN2 GIA 线路**：[搬瓦工 BandwagonHost](https://bandwagonhost.com)、[DMIT](https://www.dmit.io)、HostDare、GigsGigsCloud 等 |
| **一个域名** | 任意后缀（.com / .org / .net…） | [Namecheap](https://www.namecheap.com)、[Cloudflare](https://www.cloudflare.com/products/registrar/)、[Porkbun](https://porkbun.com) 等海外注册商 |
| **10 分钟** | 全程复制粘贴，不需要会编程 | — |

> 💡 **按人数选内存**（日常同时活跃、关联邦、纯聊天+文件）：**1GB** ≈ 几十人（建议关通话）· **2GB** ≈ 300 人 · **4GB / 2核** ≈ 500 人。tuwunel 很省，算力基本不是瓶颈。
> 💡 **磁盘才是大文件的真变量**：媒体存在服务器本地盘，E2EE 下无法去重，重度传大文件月增可达几百 GB ~ TB 级。系统盘建议 ≥ 50–100GB，重度用户请单独规划大盘或对象存储，并留意菜单里的「清理磁盘」。
> 💡 **为什么选海外商家 + CN2 GIA？** 无需实名备案、不受国内监管；CN2 GIA 从国内访问延迟低、晚高峰不卡。普通便宜 VPS 也能用，质量看运气。
> 💡 买服务器时系统镜像务必选 **Ubuntu 22.04 / 24.04**。内存小于 2.5GB 时脚本会自动加虚拟内存（swap）。

## 第 0 步：准备操作系统（买对了就跳过）

只支持 **Ubuntu 22.04 / 24.04**（或 Debian 11+）。

- **新买服务器**：下单时「操作系统 / OS / Image」选 **Ubuntu 22.04 x86_64**（或 24.04）
- **系统不对**：服务商控制台找「重装系统 / Reinstall / Install new OS」重装 Ubuntu 22.04
  - 搬瓦工：KiwiVM → **Install new OS**；DMIT：控制面板 → **Reinstall**
  - ⚠️ 重装会**清空全部数据**，老服务器先备份
- 装错也不怕：脚本检测到不支持的系统会用中文明确告诉你，不会乱装

## 第 1 步：给域名添加解析记录（约 2 分钟）

登录你**买域名的网站**，找到「DNS 解析 / DNS Records」，添加 **A 记录**。

假设域名 `mychat.org`、服务器公网 IP `1.2.3.4`：

| 记录类型 | 主机记录（名称） | 记录值 | 用途 |
|---|---|---|---|
| A | `@`（主域名） | `1.2.3.4` | 主域名 + 网页版 + 委派 |
| A | `matrix` | `1.2.3.4` | 聊天服务器 |
| A | `admin` | `1.2.3.4` | 网页管理后台（关后台可省） |
| A | `livekit` | `1.2.3.4` | 通话（关通话可省） |
| A | `matrix-rtc` | `1.2.3.4` | 通话（关通话可省） |

> 📌 主机记录只填 `matrix` 这样的**前缀**，不要填完整域名。
> 📌 添加后 1~10 分钟生效；没生效也没关系，脚本会自动等。
> 📌 只用聊天+文件（关通话）时只需 `@` / `matrix` / `admin` 三条。
> ⚠️ **用 Cloudflare 的注意**：这几条记录必须是**灰色云（仅 DNS）**，**不能开橙色云（代理）**。橙色云会：① 把大文件卡在 100MB 上限（免费/Pro）② 让证书签不出 ③ 挡住通话。安装时脚本会问你是否用了 CDN，用了就选是（放宽检查），但 `matrix` 主机务必保持灰云。

## 第 2 步：放行端口（约 1 分钟）

服务商控制台「安全组 / 防火墙」放行：

| 端口 | 协议 | 用途 |
|---|---|---|
| 80 | TCP | 申请 HTTPS 证书 |
| 443 | TCP + UDP | 网页与加密通信 |
| 7881 | TCP | 通话备用通道（关通话可省） |
| **7882** | **UDP** | **通话音视频（最容易漏！漏了 = 通话没声没画面）** |

> 📌 没有"安全组"这层的服务商直接跳过 —— 脚本会自动配系统防火墙。

## 第 3 步：连接服务器（约 1 分钟）

打开终端（Mac：「终端」；Windows：「PowerShell」），输入（IP 换成你的），回车后输服务器密码（**输密码时屏幕不显示字符是正常的**）：

```bash
ssh root@你的服务器IP
```

首次连接问 `Are you sure...`，输 `yes` 回车。

> 📌 默认用户不是 root（如 `ubuntu`）就写 `ssh ubuntu@IP`，脚本会自动提权。

## 第 4 步：运行安装命令（约 5-10 分钟，全自动）

连上服务器后，**整段复制**粘贴进终端回车：

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

接下来跟着**中文向导**走。向导会问 **6 个选项 + 是否用 CDN**，每个都有大白话说明，看不懂就一路回车（推荐值即最安全组合）：

| 选项 | 回车（推荐） | 说明 |
|---|---|---|
| 1 谁能注册 | **需邀请码** | 只有拿到你发的码的人能注册；首个注册者=管理员。（另可选完全开放，商用勿选） |
| 2 联邦互通 | **关闭** | 孤岛，外人无法给成员发消息，攻击面最小 |
| 3 语音视频通话 | **先关** | 需再加 `livekit`/`matrix-rtc` 两条 DNS 和 7881/7882 端口；先跑稳聊天+大文件 |
| 4 网页客户端 | **开启** | 成员打开 `https://你的域名` 直接注册登录，不用装 App |
| 5 Web 管理后台 | **开启** | 在 `admin.你的域名` 放一个图形管理面板 Ketesa |
| 6 单文件上限 | **4G** | 设多大都行（如 10G）；越大越占磁盘 |
| ＋ 用了 Cloudflare/CDN 吗 | **否** | 用了才选是（放宽 DNS 检查）；`matrix` 仍须灰云 |

> 🔒 **默认自动开启（不会问你，装完即有）**：① 新房间**强制端到端加密** ② **元数据最小化**（真实 IP 不入库、撤回即真删、关在线状态、日志不记 IP） ③ **Element X 手机自助注册**（原生 OIDC，注册仍需邀请码）。想关元数据加固：`PRIVACY=0 sudo -E tuwunel config`。
> 🤖 **想跳过询问全自动**：`REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

之后全自动：检测 DNS → 装 Docker → 系统调优 → 防火墙 → 生成配置 → 启动服务 → 签 HTTPS 证书 → **自动创建管理员并打印账号密码**。

看到这个就是成功：

```

 (注册[token] · 联邦[关] · 通话[关] · 网页客户端[开] · 管理后台[开] · 手机App注册[开] · 大文件[4G] · 引擎 tuwunel/Rust,免Postgres)

 成员注册 / 登录
```

> 🔑 **把管理员账号密码 + 邀请码抄下来！** 全部存在服务器 `/opt/tuwunel/CREDENTIALS.txt`（`cat /opt/tuwunel/CREDENTIALS.txt` 查看）。
> ⚠️ 结尾显示"还没就绪"多半是云安全组没放行 80/443 或 DNS 没全球生效，Caddy 会自动重试证书，无需重装。

## 👥 成员怎么加入（三选一）

1. **管理员建号（最可控）**：`sudo tuwunel adduser` 一条命令建号并设密码，把「域名 + 用户名 + 密码」发给成员，成员用 Element X 登录。邀请码全程不外流。
2. **发邀请码，网页自助注册**：把邀请码（在 `CREDENTIALS.txt`）+ `https://你的域名` 发给成员，他们在浏览器/手机浏览器自助注册。
3. **发邀请码，Element X 自助注册**：成员装 Element X → 服务器填你的域名 → 创建账户 → 输邀请码。（依赖已开启的手机注册）

## 🖥️ 图形化管理后台（Ketesa，推荐）

安装时（选项 5 开启）会自动部署一个**图形化 Web 管理后台**，浏览器点鼠标就能管，不用敲命令。

**怎么进：** 浏览器打开 `https://admin.你的域名` → **直接用管理员账号 `admin` + 密码登录**（无需额外门禁密码，面板已锁定到你的服务器）。

**能做：** 管理用户（建号/停用/改密码）、**发和吊销邀请码**、看房间/媒体、房间目录、计划任务。

> 📌 面板里「举报事件 / 被举报用户」两页可能显示报错或空白 —— 这是 tuwunel 未实现该功能，**正常**，你几乎用不到。
> 📌 面板依赖 tuwunel v1.8.1+（`:latest` 镜像已含）。若面板登录报错，先 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`。
> 📌 老服务器补装后台：`sudo tuwunel enable-admin`（需先加 `admin` 的 DNS）。

装好后，**再次运行 `sudo tuwunel` = 打开中文管理菜单**，什么命令都不用记：

```bash
sudo tuwunel
```

```
┌──────────────────────────────────────────────┐
│  tuwunel 管理菜单   你的域名                   │
└──────────────────────────────────────────────┘
  1) 查看运行状态          6) 升级服务镜像
  2) 添加团队成员          7) 清理磁盘
  3) 修改配置              8) 重启所有服务
  4) 开/关 Web 管理后台    9) 更新脚本 + 应用新功能
  5) 立即备份             10) 彻底卸载
  p) 隐私加固 / 元数据清理
  s) 涂销明文凭据文件(抗取证)
```

也可直接用命令：

```bash
sudo tuwunel adduser          # 加成员
sudo tuwunel config           # 改配置(回车=保持,数据账号不动)
sudo tuwunel update           # 从 GitHub 拉最新脚本并应用新功能(数据不动)
```

## 🔒 隐私 / 抗取证（本版重点）

这套系统为保护商业机密而设计，**默认就把服务器上的痕迹压到最小**：

- ✅ **消息内容端到端加密** —— 连你（服务方）和 VPS 商家都读不到。这是唯一和"服务器好坏"无关的硬保证。
- ✅ **元数据最小化（默认开）**：真实客户端 IP 不入库（`ip_source`）、**撤回即真删**（不再默认保留 60 天原文）、关在线状态、日志级别降到不记 IP、收紧资料/房间目录暴露面。
- ✅ **抗取证工具**：`sudo tuwunel forget-secrets` 涂销磁盘上的明文密码/邀请码；`sudo tuwunel autobackup` 备份 AES-256 加密。

**重要说明：**

- ❌ **删不掉的元数据**：房间成员关系、事件时间线、账号存在性、房间名/文件名 —— 服务器要运行就必须保留这些。E2EE 保护"内容"，**不保护"元数据"**（谁跟谁、何时、群名、文件名）。
- ❌ **磁盘被物理取得/镜像**：普通 VPS 上服务器数据是明文落盘的，被取证可提取元数据。防这一层需要选可信商家/管辖地，或进阶的全盘加密/前置盾机（非自动、需技术投入）。
- ❌ **用 Cloudflare 藏服务器 IP 基本无效**（证书透明度日志、灰云子域会泄漏）。真要藏 IP 需自建 WireGuard 前置盾机。

> 给客户的准确话术：**"聊天内容和附件端到端加密，服务方也无法读取；服务器会保留通信元数据（谁在哪个群、何时、文件名、群名），这部分在服务器被物理取得时可能被提取。"**

## 💾 备份（强烈建议）

数据库和密钥丢了**无法恢复**。本版无 PostgreSQL，备份就是打包 `data/tuwunel`（含数据库+媒体）+ `tuwunel.toml` + `.env`。

**推荐：开启自动加密备份**（每周自动、AES-256 加密、自动轮转、满盘自动跳过）：

```bash
sudo tuwunel autobackup     # 按提示设目录/保留数/频率;开启时会显示密钥,务必抄进密码管理器
```

**或手动立即备份**（菜单选 5，可设加密口令）。

**下载到本地**（在你自己电脑执行）：

```bash
scp root@你的服务器IP:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **备份密钥务必抄走**：没有密钥，加密备份永久打不开。密钥只存在服务器上，服务器没了又没抄下 = 备份全废。
> 💡 本机 `backups/` 会随服务器一起消失 —— 请定期把 `.enc` 复制到别处，或把备份目录设成挂载的外部盘/对象存储。

## ❓ 常见问题

| 问题 | 原因与解决 |
|---|---|
| **手机 Element X 连不上 / "无法连接此家伺服器"** | **首先检查手机日期时间是否准确**（设置→日期和时间→打开"自动")—— 时间不对会导致证书验证失败，这是最常见原因（尤其安卓）。其次：换网络试试、确认能在手机浏览器打开 `https://你的域名/.well-known/matrix/client`、更新 Element X 到最新版 |
| **Element X 报"需要升级以支持认证服务"** | 它在说注册。确认已开手机注册（`sudo tuwunel enable-elementx`）且 tuwunel 是最新版（`docker compose pull tuwunel`）。或改用网页注册 / 管理员 `adduser` 建号，成员用 Element X **密码登录** |
| **大文件传不上** | 检查单文件上限（`MAX_UPLOAD=10G sudo -E tuwunel config`）；用了 Cloudflare 的话 **`matrix` 主机必须灰云**（橙云 100MB 上限会掐死大文件） |
| **网页后台 admin.域名 打不开 / 登录报错** | 多半是没加 `admin` 的 DNS，或证书没签好；补记录等几分钟。登录报错先 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d` |
| 忘记 admin 密码 | `cat /opt/tuwunel/CREDENTIALS.txt`；或 `sudo tuwunel adduser` 建个新管理员 |
| 想改配置（开/关通话、注册方式、大文件、后台） | 不用重装：`sudo tuwunel config`，回车=保持，数据账号全不动 |
| 通话接通但没声没画面 | 99% 是 **7882/UDP** 没放行，去服务商安全组补上 |
| 磁盘快满了 | 菜单选 7「清理磁盘」；大文件用户请单独规划大盘或对象存储 |
| 加密房老消息"无法解密" | 正常：新设备没有历史密钥。在旧设备上对新设备做"验证会话"即可 |
| 想彻底卸载 / 重装 | `sudo tuwunel uninstall`（双重确认后才删）；重装 = 卸载后重跑安装命令 |

## 📦 服务端组件

装好后运行以下组件，全部 Docker 编排、开源可审计：

```
Caddy(自动 HTTPS) + tuwunel(Rust,内置 RocksDB,免 PostgreSQL)
  + Element Web(自家网页客户端,可选)
  + Ketesa(图形管理后台,可选)
  + LiveKit + lk-jwt-service(通话,可选)
```

安装目录 `/opt/tuwunel`，全部逻辑都在单个脚本 `matrix-tuwunel-installer.sh` 里，可自行审计。

## 🆚 和 Synapse 版有什么不同？

仓库里还有一个 Synapse 版（`matrix-installer.sh`）。简单说：

- **tuwunel 版（本文）**：Rust、免 PostgreSQL、**更省资源、大文件更强**、2GB 撑 300 人。适合大多数团队、尤其重度传大文件的场景。
- **Synapse 版**：Python + PostgreSQL，生态最成熟、管理面板最老牌，但更吃内存、大文件不如 tuwunel。

**新部署推荐 tuwunel 版。**
