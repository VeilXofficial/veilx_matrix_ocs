# 私有 Matrix 通訊伺服器 · 一鍵部署（tuwunel 版）

**你嘅伺服器、你嘅資料 —— 為機密性同資料主權而設嘅自架團隊通訊系統。**

本版採用 **tuwunel**（Rust 引擎、內建資料庫、**免 PostgreSQL**）：更慳資源、更穩定，**可以好似 Telegram 咁傳大檔案／大圖／長片**。2GB 記憶體就可以撐起一個中型團隊。端對端加密、中繼資料最小化、邀請制註冊全部**預設開啟**。一行指令部署 —— 唔使做工程師都用得，每個選項都有白話解釋。

推薦用戶端：**Element X**。專屬嘅 **VeilX** 用戶端開發緊（更美觀易用、功能更多、開源可稽核；營運團隊設於英國、新加坡、日本等地）。客戶資料、合約報價、內部討論、語音視像會議 —— 全部只存喺你自己嘅伺服器上。基於開放嘅 [Matrix](https://matrix.org) 協定，原始碼公開可稽核（非商業免費）。

## 📋 開始前嘅準備（3 樣）

| 需要乜 | 要求 | 去邊度 |
|---|---|---|
| **一台雲端主機（VPS）** | Ubuntu 22.04／24.04（或 Debian 11+），有公網 IP。**按團隊人數揀記憶體**（見下）。**要傳大檔案就要預留足夠磁碟。** | [Vultr](https://www.vultr.com)、[Linode（Akamai）](https://www.linode.com)、[DigitalOcean](https://www.digitalocean.com)、[Hetzner](https://www.hetzner.com/cloud) 等你信得過嘅商家 |
| **一個網域** | 咩結尾都得（.com／.org／.net…） | [Namecheap](https://www.namecheap.com)、[Porkbun](https://porkbun.com)、[Gandi](https://www.gandi.net) 等 |
| **10 分鐘** | 全程複製貼上，唔使寫程式 | — |

> 💡 **按同時在線人數揀記憶體**（同時活躍、閂聯邦、純傾偈＋檔案）：**1GB** ≈ 幾十人（建議閂通話）· **2GB** ≈ 300 人 · **4GB／2 vCPU** ≈ 500 人。tuwunel 好慳，算力通常唔係樽頸。
> 💡 **磁碟先係大檔案嘅真變數。** 媒體存喺伺服器本機磁碟，而且因為端對端加密無法去重，重度傳大檔案每月可以增加幾百 GB～TB。系統碟建議 ≥ 50–100GB，重度用戶請另備大容量磁碟或物件儲存，並留意選單裡面嘅「清理磁碟」。
> 💡 **管轄地對隱私有影響。** 揀一個你放心嘅商家同國家 —— 主機商原則上可以被強制配合。端對端加密無論如何都保護訊息「內容」，但中繼資料存喺主機上。
> 💡 落單時務必揀 **Ubuntu 22.04／24.04** 映像檔。記憶體低過 2.5GB 時腳本會自動加虛擬記憶體（swap）。

## 第 0 步：準備作業系統（買啱咗就跳過）

只支援 **Ubuntu 22.04／24.04**（或 Debian 11+）。

- **新主機**：落單時「作業系統／OS／Image」揀 **Ubuntu 22.04 x86_64**（或 24.04）。
- **系統唔啱**：用商家控制台嘅「重裝／Reinstall／Rebuild」裝 Ubuntu 22.04。⚠️ 呢個會**清空所有資料**，舊主機記得先備份。
- 唔肯定而家嘅系統？連上之後（第 3 步）執行 `cat /etc/os-release`。
- 裝錯都唔使驚：腳本偵測到唔支援嘅系統會用中文清楚話你知，唔會亂裝。

## 第 1 步：加 DNS 記錄（約 2 分鐘）

登入你**買網域嗰個網站**嘅 DNS 設定，加 **A 記錄**。

假設網域 `mychat.org`、主機公網 IP `1.2.3.4`：

| 類型 | 主機（名稱） | 值 | 用途 |
|---|---|---|---|
| A | `@`（主網域） | `1.2.3.4` | 主網域＋網頁版＋委派 |
| A | `matrix` | `1.2.3.4` | 傾偈伺服器 |
| A | `admin` | `1.2.3.4` | 網頁管理後台（閂後台可省） |
| A | `livekit` | `1.2.3.4` | 通話（閂通話可省） |
| A | `matrix-rtc` | `1.2.3.4` | 通話（閂通話可省） |

> 📌 主機欄只填 `matrix` 呢種**前綴**，唔好填完整網域。
> 📌 記錄 1～10 分鐘生效；未生效都唔緊要，安裝程式會自動等。
> 📌 淨係傾偈＋檔案（閂通話）就只需 `@`／`matrix`／`admin` 三條。
> ⚠️ **用 Cloudflare 嘅注意**：呢幾條記錄必須係**灰色雲（只 DNS）**，**唔可以開橙色雲（代理）**。橙色雲會 ① 將大檔案卡喺 100MB 上限（免費／Pro）② 令憑證簽唔到 ③ 阻住通話。安裝時腳本會問你係咪用咗 CDN（用咗就揀係，放寬檢查），但 `matrix` 主機務必保持灰色雲。

## 第 2 步：開放防火牆連接埠（約 1 分鐘）

喺商家控制台嘅「安全群組／防火牆」開放：

| 連接埠 | 協定 | 用途 |
|---|---|---|
| 80 | TCP | 申請 HTTPS 憑證 |
| 443 | TCP + UDP | 網頁同加密通訊 |
| 7881 | TCP | 通話備用通道（閂通話可省） |
| **7882** | **UDP** | **通話音訊視像（最易漏！漏咗＝通話無聲無畫面）** |

> 📌 有啲商家無「安全群組」呢層設定就直接跳過 —— 腳本會自動設定系統防火牆。

## 第 3 步：連線到你嘅主機（約 1 分鐘）

打開終端機（macOS：「終端機」；Windows：「PowerShell」），輸入以下指令（IP 換成你嘅），回車後輸入主機密碼（**輸入密碼時螢幕唔顯示任何字元係正常嘅**）：

```bash
ssh root@你嘅主機IP
```

第一次連線問 `Are you sure...` 時輸入 `yes` 回車。

> 📌 預設使用者唔係 root（例如 `ubuntu`）就寫 `ssh ubuntu@IP`，腳本會自動提權。

## 第 4 步：執行安裝指令（約 5–10 分鐘，全自動）

連上之後，**成段複製**貼入終端機回車：

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

跟住中文精靈行。會問 **6 個選項＋係咪用 CDN**，每個都有白話解釋，睇唔明就一路回車（預設值即最安全組合）：

| 選項 | 回車（推薦） | 說明 |
|---|---|---|
| 1 邊個可以註冊 | **需邀請碼** | 只有攞到你發嘅碼嘅人先可以註冊；首位註冊者＝管理員。（另可揀完全開放，商用勿選） |
| 2 聯邦互通 | **閂** | 孤島模式，外人無法對成員發訊息，攻擊面最細。 |
| 3 語音視像通話 | **先閂** | 需再加 `livekit`／`matrix-rtc` 兩條 DNS 同 7881／7882 連接埠；先跑穩傾偈＋大檔案。 |
| 4 網頁用戶端 | **開** | 成員打開 `https://你嘅網域` 直接註冊登入，免裝 App。 |
| 5 網頁管理後台 | **開** | 喺 `admin.你嘅網域` 放一個圖形管理後台 Ketesa。 |
| 6 單檔上限 | **4G** | 設幾大都得（例如 10G）；越大越佔磁碟。 |
| ＋ 係咪用咗 Cloudflare／CDN | **否** | 用咗先揀係（放寬 DNS 檢查）；`matrix` 仍須灰色雲。 |

> 🔒 **自動開啟（唔會問你，裝完即有）**：① 新房間**強制端對端加密** ② **中繼資料最小化**（真實 IP 唔入庫、撤回即真刪、閂在線狀態、日誌唔記 IP） ③ **Element X 手機自助註冊**（原生 OIDC，註冊仍需邀請碼）。想閂中繼資料加固：`PRIVACY=0 sudo -E tuwunel config`。
> 🤖 **全自動唔詢問**：`REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

之後全自動：偵測 DNS → 安裝 Docker → 系統調校 → 防火牆 → 產生設定 → 啟動服務 → 申請 HTTPS 憑證 → **自動建立管理員並印出帳號密碼**。

見到呢個就係成功：

```
========================================================
 🎉 tuwunel 部署完成!  mychat.org
 (註冊[token] · 聯邦[閂] · 通話[閂] · 網頁[開] · 後台[開] · 手機註冊[開] · 大檔案[4G] · 引擎 tuwunel/Rust)

 成員註冊 / 登入
   網頁版(推薦):瀏覽器打開 https://mychat.org 直接註冊登入
   手機 App:裝 Element X → 伺服器填 mychat.org → 可註冊(需邀請碼)或登入

 管理員(已自動建立)
   帳號: admin    密碼: xxxxxxxxxxxx
   網頁管理後台: https://admin.mychat.org  (用上面嘅管理員帳號密碼登入)

 日常管理:  sudo tuwunel   (選單)    sudo tuwunel adduser   (加成員)
========================================================
```

> 🔑 **將管理員帳號密碼＋邀請碼抄低！** 全部存喺主機嘅 `/opt/tuwunel/CREDENTIALS.txt`（`cat /opt/tuwunel/CREDENTIALS.txt` 睇）。
> ⚠️ 結尾顯示「仲未就緒」多數係雲端安全群組無放行 80／443，或者 DNS 未全球生效。Caddy 會自動重試憑證，唔使重裝。

## 👥 成員點加入（三揀一）

1. **管理員起帳號（最可控）**：`sudo tuwunel adduser` 一條指令起帳號兼設密碼，將「網域＋用戶名＋密碼」發俾成員。邀請碼全程唔外流。
2. **發邀請碼，網頁自助註冊**：將邀請碼（喺 `CREDENTIALS.txt`）＋ `https://你嘅網域` 發俾成員，佢哋喺瀏覽器自助註冊。
3. **發邀請碼，Element X 自助註冊**：成員裝 Element X → 伺服器填你嘅網域 → 建立帳號 → 輸入邀請碼。

## 🖥️ 圖形化管理後台（Ketesa，推薦）

安裝時（選項 5 開啟）會自動部署一個**圖形化網頁管理後台**，用滑鼠㩒就管到，唔使打指令。

**點入去：** 瀏覽器打開 `https://admin.你嘅網域` → **直接用管理員帳號 `admin` ＋密碼登入**（唔需額外門禁密碼，後台已鎖定到你嘅伺服器）。

**做到啲乜：** 管理用戶（起號／停用／改密碼）、**發放同撤銷邀請碼**、睇房間／媒體、房間目錄、排程工作。

> 📌 後台裡面「檢舉事件／被檢舉用戶」兩頁可能報錯或空白 —— 呢個係 tuwunel 未實作嗰個功能，**正常**，你幾乎用唔到。
> 📌 後台需要 tuwunel v1.8.1+（`:latest` 映像已含）。若後台登入報錯，先 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`。
> 📌 舊伺服器補裝後台：`sudo tuwunel enable-admin`（需先加 `admin` 嘅 DNS）。

## 🔧 日常維護:管理選單

裝好之後，**再次執行 `sudo tuwunel` = 打開中文管理選單**，咩指令都唔使記：

```bash
sudo tuwunel
```

```
┌──────────────────────────────────────────────┐
│  tuwunel 管理選單   你嘅網域                   │
└──────────────────────────────────────────────┘
  1) 檢視執行狀態          6) 升級服務映像
  2) 新增團隊成員          7) 清理磁碟
  3) 修改設定              8) 重啟所有服務
  4) 開/閂 網頁管理後台    9) 更新腳本 + 套用新功能
  5) 立即備份             10) 徹底解除安裝
  p) 隱私加固 / 中繼資料清理
  s) 塗銷明文憑證檔(抗鑑識)
  b) 自動定時加密備份
  0) 離開
```

亦可以直接用指令：

```bash
sudo tuwunel adduser          # 加成員
sudo tuwunel config           # 改設定(回車=保持,資料帳號唔郁)
sudo tuwunel update           # 從 GitHub 拉最新腳本並套用新功能(資料唔郁)
sudo tuwunel enable-admin     # 舊伺服器補裝網頁管理後台
sudo tuwunel enable-elementx  # 開 Element X 手機自助註冊
sudo tuwunel autobackup       # 開啟每週自動加密備份
sudo tuwunel privacy          # 隱私/中繼資料:睇可以刪啲乜、清日誌
sudo tuwunel forget-secrets   # 抗鑑識:塗銷磁碟上嘅明文密碼/邀請碼
sudo tuwunel uninstall        # 解除安裝
```

> 🔄 **改設定唔使重裝**（選單 3）：重問選項，**回車＝保持目前值**，改完自動重啟，帳號／記錄／金鑰全部唔郁。

## 🔒 隱私 / 抗鑑識（本版重點）

本系統為保護機密通訊而設計，**預設就將伺服器上嘅痕跡壓到最細**：

- ✅ **訊息內容端對端加密** —— 連你（營運方）同主機商都讀唔到。呢個係唯一同「伺服器好壞」無關嘅硬保證。
- ✅ **中繼資料最小化（預設開）**：真實用戶端 IP 唔入庫（`ip_source`）、**撤回即真刪**（唔再預設保留 60 日原文）、閂在線狀態、日誌等級低到唔記 IP、收緊個人資料／房間目錄嘅曝露面。
- ✅ **抗鑑識工具**：`sudo tuwunel forget-secrets` 塗銷磁碟上嘅明文密碼／邀請碼；`sudo tuwunel autobackup` 備份採 AES-256 加密。

**誠實嘅邊界（務必對客戶如實講，唔好過度承諾）：**

- ❌ **刪唔到嘅中繼資料**：房間成員關係、事件時間軸、帳號存在性、房間名稱／檔案名稱 —— 伺服器要運作就必須保留呢啲。端對端加密保護「內容」，**唔保護中繼資料**（邊個同邊個、幾時、房間名、檔名）。
- ❌ **磁碟被實體攞到／鏡像**：普通 VPS 上資料係明文落盤，被鑑識可以提取中繼資料。防呢層需要揀可信商家／管轄地，或者進階嘅全碟加密／前置盾機（唔係自動，需技術投入）。
- ❌ **用 Cloudflare 藏伺服器 IP 基本無效**（憑證透明度日誌、灰色雲子網域會洩漏）。真要藏 IP 需自架 WireGuard 前置盾機。

> 俾客戶嘅準確說法：**「傾偈內容同附件端對端加密，營運方都無法讀取；伺服器會保留通訊中繼資料（邊個喺邊個房間、幾時、檔名、房間名），呢部分喺伺服器被實體攞到時可能被提取。」**

## 💾 備份（強烈建議）

資料庫同金鑰唔見咗就**無法還原**。本版無 PostgreSQL，備份就係打包 `data/tuwunel`（資料庫＋媒體）＋ `tuwunel.toml` ＋ `.env`。

**推薦:開啟自動加密備份**（每週自動、AES-256、自動輪替、磁碟將滿時自動略過）：

```bash
sudo tuwunel autobackup     # 按提示設目錄/保留數/頻率;開啟時會顯示金鑰,務必抄入密碼管理器
```

**或者立即備份**（選單揀 5，可設加密密語）。

**下載到本機**（喺你自己電腦執行）：

```bash
scp root@你嘅主機IP:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **備份金鑰務必抄走**：無咗金鑰，加密備份永遠打唔開。金鑰只存喺伺服器上，伺服器無咗你又無抄低 = 備份全廢。
> 💡 本機 `backups/` 會隨伺服器一齊消失 —— 請定期將 `.enc` 複製去第度，或者將備份目錄設成掛載嘅外接磁碟／物件儲存。

## ❓ 常見問題

| 問題 | 原因同解決 |
|---|---|
| **手機 Element X 連唔上／「無法連線至此家伺服器」** | **先檢查手機日期時間啱唔啱**（設定→日期同時間→打開「自動」）—— 時間唔啱會導致憑證驗證失敗，呢個係最常見原因（尤其 Android）。跟住：換個網路試、確認可以喺手機瀏覽器打開 `https://你嘅網域/.well-known/matrix/client`、更新 Element X 到最新版。 |
| **Element X 話「需要升級以支援認證服務」** | 嗰個係講註冊。確認已開手機註冊（`sudo tuwunel enable-elementx`）而且 tuwunel 係最新版（`docker compose pull tuwunel`）。或者改用網頁註冊／管理員 `adduser` 起帳號，成員用 Element X **密碼登入**。 |
| **大檔案傳唔上** | 檢查單檔上限（`MAX_UPLOAD=10G sudo -E tuwunel config`）；用 Cloudflare 嘅話 **`matrix` 主機必須灰色雲**（橙色雲 100MB 上限會卡死大檔案）。 |
| **網頁後台 admin.網域 打唔開／登入報錯** | 多數係無加 `admin` 嘅 DNS，或者憑證仲未簽好；補記錄等幾分鐘。登入報錯先 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`。 |
| 唔記得 admin 密碼 | `cat /opt/tuwunel/CREDENTIALS.txt`；或 `sudo tuwunel adduser` 起個新管理員。 |
| 想改設定（通話、註冊方式、大檔案、後台） | 唔使重裝：`sudo tuwunel config`，回車＝保持，資料帳號全部唔郁。 |
| 通話接通但無聲無畫面 | 99% 係 **7882/UDP** 無放行，去商家安全群組補上。 |
| 磁碟就快滿 | 選單揀 7「清理磁碟」；大檔案用戶請另備大容量磁碟或物件儲存。 |
| 加密房間舊訊息「無法解密」 | 正常：新裝置無歷史金鑰。喺舊裝置對新裝置做「驗證工作階段」即可。 |
| 想徹底解除安裝／重裝 | `sudo tuwunel uninstall`（雙重確認後先刪）；重裝 = 解除安裝後重跑安裝指令。 |

## 📦 伺服器端組件

裝好後執行以下組件，全部以 Docker 編排、開源可稽核：

```
Caddy(自動 HTTPS) + tuwunel(Rust,內建 RocksDB,免 PostgreSQL)
  + Element Web(自家網頁用戶端,可選)
  + Ketesa(圖形管理後台,可選)
  + LiveKit + lk-jwt-service(通話,可選)
```

安裝目錄 `/opt/tuwunel`，全部邏輯都喺單一腳本 `matrix-tuwunel-installer.sh` 裡面，可以自行稽核。

## 🆚 tuwunel 版 vs Synapse 版

倉庫裡面仲有一個 Synapse 版（`matrix-installer.sh`）。簡單講：

- **tuwunel 版（本文）**：Rust、免 PostgreSQL、**更慳資源、大檔案更強**、2GB 撐 300 人。適合大多數團隊、尤其重度傳大檔案嘅場景。
- **Synapse 版**：Python + PostgreSQL，生態最成熟、管理面板最老牌，但更食記憶體、大檔案不如 tuwunel。

**新部署推薦 tuwunel 版。**
