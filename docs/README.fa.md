# سرور ارتباطی خصوصی Matrix · استقرار با یک فرمان (نسخهٔ tuwunel)

**سرور شما، دادهٔ شما — یک پیام‌رسان تیمیِ خودمیزبان، ساخته‌شده برای محرمانگی و حاکمیت داده.**

</div>

<div dir="rtl">

بر پایهٔ موتور **tuwunel** (نوشته‌شده با Rust، پایگاه‌دادهٔ داخلی، **بدون نیاز به PostgreSQL**): سبک‌تر، پایدارتر و قادر به **ارسال فایل‌های بزرگ / عکس / ویدیوی طولانی مانند Telegram**. با ۲ گیگابایت رم یک تیم متوسط را به‌راحتی اجرا می‌کند. رمزنگاری سرتاسری (E2E)، کمینه‌سازی فراداده و ثبت‌نامِ فقط با دعوت به‌صورت **پیش‌فرض فعال** هستند. استقرار تنها با یک فرمان — لازم نیست مدیر سیستم باشید؛ هر گزینه توضیح ساده دارد.

کلاینت پیشنهادی: **Element X**. کلاینت اختصاصی **VeilX** در حال توسعه است (تمیزتر، ساده‌تر، با امکانات بیشتر؛ متن‌باز و قابل ممیزی؛ تیم عملیات در بریتانیا، سنگاپور و ژاپن). پرونده‌های مشتریان، قراردادها، بحث‌های داخلی، جلسات صوتی و تصویری — همه فقط روی سرور خودتان باقی می‌مانند. مبتنی بر پروتکل باز [Matrix](https://matrix.org). کد منبع عمومی و قابل ممیزی (برای استفادهٔ غیرتجاری رایگان).

## ✨ پس از نصب چه چیزی به‌دست می‌آورید

- 💬 گفتگوی متنی و اتاق‌های گروهی (**رمزنگاری سرتاسری — نه سرور و نه شرکت میزبان محتوای پیام‌ها را نمی‌بینند**)
- 📁 **ارسال فایل‌های بزرگ / عکس / ویدیوی طولانی** (سقف پیش‌فرض هر فایل **۴ گیگابایت**، قابل افزایش — ویژگی اصلی)
- 📞 تماس صوتی / تصویری دونفره و گروهی (اختیاری)
- 📱 ثبت‌نام از گوشی: **Element X** را نصب کنید، دامنه را وارد کنید، ثبت‌نام و ورود (بدون مراجعه به element.io)
- 🌐 کلاینت وب روی دامنهٔ خودتان: در مرورگر `https://دامنهٔ‌شما` را باز کنید تا ثبت‌نام/ورود کنید، بدون نیاز به اپ
- 🖥️ **پنل مدیریت وب گرافیکی** (Ketesa): مدیریت کاربران، صدور/ابطال کد دعوت، مشاهدهٔ اتاق‌ها و رسانه‌ها
- 🔒 **سخت‌سازیِ حریم خصوصی به‌صورت پیش‌فرض**: IP واقعی کاربر ذخیره نمی‌شود، حذف‌ها دائمی‌اند، وضعیت حضور خاموش، لاگ‌ها IP ثبت نمی‌کنند
- 👥 فقط با دعوت: به‌صورت پیش‌فرض تنها کسی که کد دعوت دارد می‌تواند ثبت‌نام کند
- ⚡ **کم‌مصرف**: یک فرایند Rust، بدون PostgreSQL — ۲ گیگابایت رم برای حدود ۳۰۰ نفر

## 📋 پیش از شروع (۳ چیز)

| چه چیزی | نیازمندی | از کجا |
|---|---|---|
| **یک سرور ابری (VPS)** | Ubuntu 22.04 / 24.04 (یا Debian 11+)، با IP عمومی. **رم را بر اساس اندازهٔ تیم انتخاب کنید** (پایین‌تر). **اگر فایل بزرگ به‌اشتراک می‌گذارید، دیسک کافی تدارک ببینید.** | [Vultr](https://www.vultr.com)، [DigitalOcean](https://www.digitalocean.com)، [Hetzner](https://www.hetzner.com/cloud)، [OVHcloud](https://www.ovhcloud.com) یا هر ارائه‌دهنده‌ای که به آن اعتماد دارید |
| **یک دامنه** | هر پسوندی (.com / .org / .net…) | [Namecheap](https://www.namecheap.com)، [Porkbun](https://porkbun.com)، [Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) |
| **۱۰ دقیقه** | همه‌چیز کپی‌وچسباندن است؛ برنامه‌نویسی لازم نیست | — |

> 💡 **رم را بر اساس کاربران فعال هم‌زمان انتخاب کنید** (فدراسیون خاموش، چت + فایل): **۱ گیگ** ≈ چند ده نفر (تماس را خاموش کنید) · **۲ گیگ** ≈ ۳۰۰ نفر · **۴ گیگ / ۲ vCPU** ≈ ۵۰۰ نفر. tuwunel سبک است و CPU به‌ندرت گلوگاه می‌شود.
> 💡 **برای فایل‌های بزرگ، دیسک متغیر واقعی است.** رسانه روی دیسک محلی سرور ذخیره می‌شود و به‌دلیل رمزنگاری سرتاسری قابل حذف تکراری نیست. مصرف سنگین می‌تواند ماهانه از صدها گیگ تا ترابایت اضافه کند. دیسک سیستم ≥ ۵۰–۱۰۰ گیگ؛ برای مصرف سنگین یک حجم بزرگ یا ذخیره‌سازی شیءگرا تدارک ببینید و به گزینهٔ «پاک‌سازی دیسک» توجه کنید.
> 💡 **حوزهٔ قضایی برای حریم خصوصی مهم است.** ارائه‌دهنده و کشوری را انتخاب کنید که به آن راحت هستید — میزبان در اصل ممکن است به همکاری وادار شود. رمزنگاری سرتاسری در هر حال *محتوا* را حفظ می‌کند، اما فراداده روی میزبان می‌ماند.
> 💡 هنگام سفارش حتماً تصویر **Ubuntu 22.04 / 24.04** را انتخاب کنید. اگر رم کمتر از ۲٫۵ گیگ باشد، اسکریپت به‌صورت خودکار swap اضافه می‌کند.

## گام ۰ — آماده‌سازی سیستم‌عامل (اگر درست سفارش داده‌اید، رد شوید)

فقط **Ubuntu 22.04 / 24.04** (یا Debian 11+) پشتیبانی می‌شود.

- **سرور جدید**: هنگام سفارش در قسمت «OS / Image» گزینهٔ **Ubuntu 22.04 x86_64** (یا 24.04) را انتخاب کنید.
- **سیستم‌عامل اشتباه**: از «نصب مجدد / Reinstall / Rebuild» ارائه‌دهنده برای نصب Ubuntu 22.04 استفاده کنید. ⚠️ این کار **همهٔ داده‌ها را پاک می‌کند** — از سرور موجود ابتدا نسخهٔ پشتیبان بگیرید.
- نمی‌دانید چه اجرا می‌کنید؟ پس از اتصال (گام ۳) فرمان `cat /etc/os-release` را اجرا کنید.
- اگر روی سیستم‌عامل اشتباه نصب کنید، اسکریپت آن را تشخیص داده و به‌روشنی اطلاع می‌دهد؛ چیزی خراب نمی‌شود.

## گام ۱ — افزودن رکوردهای DNS (~۲ دقیقه)

در پنل DNS **ثبت‌کنندهٔ دامنه**، **رکوردهای A** اضافه کنید.

فرض کنیم دامنه `mychat.org` و IP سرور `1.2.3.4` است:

| نوع | میزبان (نام) | مقدار | کاربرد |
|---|---|---|---|
| A | `@` (دامنهٔ ریشه) | `1.2.3.4` | دامنهٔ ریشه + کلاینت وب + واگذاری |
| A | `matrix` | `1.2.3.4` | سرور خانگی |
| A | `admin` | `1.2.3.4` | پنل مدیریت وب (اگر مدیریت خاموش است، رد شوید) |
| A | `livekit` | `1.2.3.4` | تماس (اگر تماس خاموش است، رد شوید) |
| A | `matrix-rtc` | `1.2.3.4` | تماس (اگر تماس خاموش است، رد شوید) |

> 📌 در فیلد میزبان فقط **پیشوند** (`matrix`) را بنویسید، نه دامنهٔ کامل را.
> 📌 رکوردها در ۱ تا ۱۰ دقیقه منتشر می‌شوند؛ اگر هنوز فعال نشده‌اند نصب‌کننده خودش منتظر می‌ماند.
> 📌 برای فقط چت + فایل (تماس خاموش) تنها `@` / `matrix` / `admin` لازم است.
> ⚠️ **اگر از Cloudflare استفاده می‌کنید**: این رکوردها باید **ابر خاکستری (فقط DNS)** باشند؛ **ابر نارنجی (پروکسی) را روشن نکنید**. پروکسی: ① فایل‌های بزرگ را به ۱۰۰ مگابایت محدود می‌کند (Free/Pro) ② صدور گواهی را مختل می‌کند ③ رسانهٔ تماس را مسدود می‌کند. نصب‌کننده می‌پرسد آیا از CDN استفاده می‌کنید (اگر بله، بررسی DNS آسان می‌شود)، اما میزبان `matrix` باید ابر خاکستری بماند.

## گام ۲ — باز کردن پورت‌های فایروال (~۱ دقیقه)

در «Security Group / Firewall» ارائه‌دهنده اجازه دهید:

| پورت | پروتکل | کاربرد |
|---|---|---|
| 80 | TCP | صدور گواهی HTTPS |
| 443 | TCP + UDP | وب و ترافیک رمزنگاری‌شده |
| 7881 | TCP | کانال پشتیبان تماس (بدون تماس رد شوید) |
| **7882** | **UDP** | **صدا/تصویر تماس (بیشترین احتمال فراموشی! بدون آن = بدون صدا و تصویر)** |

> 📌 اگر ارائه‌دهنده لایهٔ «security group» ندارد، رد شوید — اسکریپت فایروال سیستم را به‌صورت خودکار تنظیم می‌کند.

## گام ۳ — اتصال به سرور (~۱ دقیقه)

ترمینال را باز کنید (macOS: «Terminal»؛ Windows: «PowerShell»)، دستور زیر را اجرا کنید (IP را جایگزین کنید) و در صورت درخواست، رمز سرور را وارد کنید (**طبیعی است که هنگام تایپ رمز، چیزی روی صفحه نمایش داده نشود**):

```bash
ssh root@IP_سرور_شما
```

در نخستین اتصال، به پرسش اثرانگشت `yes` پاسخ دهید.

> 📌 اگر کاربر پیش‌فرض root نیست (مثلاً `ubuntu`) از `ssh ubuntu@IP` استفاده کنید؛ اسکریپت خودش سطح دسترسی را بالا می‌برد.

## گام ۴ — اجرای نصب‌کننده (~۵ تا ۱۰ دقیقه، خودکار)

پس از اتصال، **کل خط را کپی** و در ترمینال بچسبانید:

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

از دستیار (wizard) پیروی کنید. **۶ گزینه + یک پرسش CDN** می‌پرسد؛ هر مورد روی صفحه توضیح دارد، پس در صورت تردید Enter بزنید (مقادیر پیش‌فرض امن‌ترین ترکیب‌اند):

| گزینه | Enter (توصیه‌شده) | توضیح |
|---|---|---|
| ۱ چه کسی ثبت‌نام کند | **نیاز به کد دعوت** | فقط دارندهٔ کد وارد می‌شود؛ نخستین ثبت‌نام‌کننده مدیر می‌شود. (یا کاملاً باز — برای کسب‌وکار مناسب نیست.) |
| ۲ فدراسیون | **خاموش** | حالت جزیره: کسی از بیرون نمی‌تواند به اعضا پیام دهد؛ کمترین سطح حمله. |
| ۳ تماس صوتی/تصویری | **ابتدا خاموش** | نیازمند دو رکورد DNS اضافه (`livekit`/`matrix-rtc`) و پورت‌های 7881/7882؛ ابتدا چت + فایل را پایدار کنید. |
| ۴ کلاینت وب | **روشن** | اعضا `https://دامنهٔ‌شما` را برای ثبت‌نام/ورود باز می‌کنند، بدون اپ. |
| ۵ پنل مدیریت وب | **روشن** | یک پنل گرافیکی (Ketesa) روی `admin.دامنهٔ‌شما`. |
| ۶ حداکثر اندازهٔ فایل | **4G** | هر مقداری (مثلاً 10G)؛ بزرگ‌تر یعنی دیسک بیشتر. |
| ＋ پشت Cloudflare/CDN هستید؟ | **خیر** | فقط اگر از CDN استفاده می‌کنید «بله» (بررسی DNS آسان می‌شود)؛ `matrix` باید ابر خاکستری بماند. |

> 🔒 **به‌صورت خودکار فعال (پرسیده نمی‌شود؛ بلافاصله پس از نصب اعمال می‌شود):** ① اتاق‌های جدید **با رمزنگاری E2E اجباری** ② **کمینه‌سازی فراداده** (IP واقعی ذخیره نمی‌شود، حذف‌ها دائمی، حضور خاموش، لاگ بدون IP) ③ **ثبت‌نام خودکار با Element X** (OIDC بومی؛ ثبت‌نام همچنان به کد دعوت نیاز دارد). برای خاموش‌کردن سخت‌سازی فراداده: `PRIVACY=0 sudo -E tuwunel config`.
> 🤖 **نصب کاملاً بدون دخالت:** `REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

سپس همه‌چیز خودکار پیش می‌رود: بررسی DNS → نصب Docker → تنظیم سیستم → فایروال → تولید پیکربندی → راه‌اندازی سرویس‌ها → دریافت گواهی HTTPS → **ساخت خودکار مدیر و چاپ حساب/رمز**.

موفقیت این‌گونه دیده می‌شود:

```
========================================================
 🎉 tuwunel مستقر شد!  mychat.org
 (ثبت‌نام[token] · فدراسیون[off] · تماس[off] · وب[on] · مدیریت[on] · ثبت‌نام‌موبایل[on] · فایل‌بزرگ[4G] · موتور tuwunel/Rust، بدون Postgres)

 ثبت‌نام / ورود اعضا
   وب (توصیه‌شده): https://mychat.org را باز کنید تا ثبت‌نام و ورود کنید
   اپ موبایل: Element X را نصب کنید → سرور "mychat.org" → ثبت‌نام (کد دعوت) یا ورود

 مدیر (به‌صورت خودکار ساخته شد)
   حساب: admin    رمز: xxxxxxxxxxxx
   پنل وب: https://admin.mychat.org  (با حساب مدیر بالا وارد شوید)

 مدیریت روزمره:  sudo tuwunel   (منو)    sudo tuwunel adduser   (افزودن عضو)
========================================================
```

> 🔑 **حساب/رمز مدیر و کد دعوت را یادداشت کنید!** همه در سرور در `/opt/tuwunel/CREDENTIALS.txt` هستند (`cat /opt/tuwunel/CREDENTIALS.txt`).
> ⚠️ پیام «هنوز آماده نیست» در پایان معمولاً یعنی فایروال ابری پورت‌های 80/443 را اجازه نمی‌دهد، یا DNS در سطح جهانی منتشر نشده. Caddy گواهی را خودکار دوباره تلاش می‌کند؛ نیازی به نصب مجدد نیست.

## 👥 اعضا چگونه می‌پیوندند (یکی را انتخاب کنید)

۱. **مدیر حساب می‌سازد (بیشترین کنترل):** `sudo tuwunel adduser` با یک فرمان کاربر می‌سازد و رمز تعیین می‌کند؛ «دامنه + نام‌کاربری + رمز» را بدهید. کد دعوت از دست شما خارج نمی‌شود.
۲. **پخش کد دعوت، ثبت‌نام خودکار وب:** کد (در `CREDENTIALS.txt`) + `https://دامنهٔ‌شما` را بدهید تا اعضا خودشان در مرورگر ثبت‌نام کنند.
۳. **پخش کد، ثبت‌نام خودکار با Element X:** Element X را نصب کنند → دامنه را وارد کنند → حساب بسازند → کد دعوت را وارد کنند.

## 🖥️ پنل مدیریت گرافیکی (Ketesa، توصیه‌شده)

اگر فعال باشد (گزینهٔ ۵)، نصب‌کننده یک **پنل مدیریت وب گرافیکی** مستقر می‌کند — همه‌چیز با ماوس، بدون فرمان.

**نحوهٔ ورود:** به `https://admin.دامنهٔ‌شما` بروید → **مستقیماً با حساب مدیر `admin` + رمز وارد شوید** (بدون رمز دروازهٔ جداگانه؛ پنل به سرور شما قفل است).

**چه کارهایی می‌توانید بکنید:** مدیریت کاربران (ساخت/غیرفعال/بازنشانی رمز)، **صدور و ابطال کد دعوت**، مشاهدهٔ اتاق‌ها/رسانه‌ها، فهرست اتاق‌ها، وظایف زمان‌بندی‌شده.

> 📌 صفحه‌های «گزارش‌ها / کاربران گزارش‌شده» ممکن است خطا دهند یا خالی باشند — tuwunel آن قابلیت را پیاده‌سازی نکرده و این **طبیعی** است؛ تقریباً هرگز به آن نیاز ندارید.
> 📌 پنل به tuwunel v1.8.1+ نیاز دارد (تصویر `:latest` دارد). اگر ورود پنل خطا داد: `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`.
> 📌 افزودن پنل به سرور موجود: `sudo tuwunel enable-admin` (ابتدا رکورد DNS `admin` را اضافه کنید).

## 🔧 مدیریت روزمره: منو

پس از نصب، **اجرای `sudo tuwunel` منوی مدیریت را باز می‌کند** (نیازی به حفظ فرمان‌ها نیست):

```bash
sudo tuwunel
```

```
┌──────────────────────────────────────────────┐
│  منوی مدیریت tuwunel   دامنهٔ‌شما             │
└──────────────────────────────────────────────┘
  1) وضعیت                    6) ارتقای ایمیج‌ها
  2) افزودن عضو               7) پاک‌سازی دیسک
  3) تغییر پیکربندی           8) راه‌اندازی مجدد سرویس‌ها
  4) روشن/خاموش پنل وب        9) به‌روزرسانی اسکریپت + قابلیت‌های جدید
  5) پشتیبان‌گیری اکنون       10) حذف کامل
  p) سخت‌سازی حریم خصوصی / پاک‌سازی فراداده
  s) ماسک‌کردن فایل اعتبارنامهٔ متن‌ساده (ضدجرم‌یابی)
  b) پشتیبان‌گیری رمزنگاری‌شدهٔ زمان‌بندی‌شدهٔ خودکار
  0) خروج
```

یا مستقیماً از فرمان‌ها استفاده کنید:

```bash
sudo tuwunel adduser          # افزودن عضو
sudo tuwunel config           # تغییر پیکربندی (Enter = حفظ؛ داده/حساب‌ها دست‌نخورده)
sudo tuwunel update           # دریافت آخرین اسکریپت از GitHub و اعمال قابلیت‌های جدید (داده دست‌نخورده)
sudo tuwunel enable-admin     # افزودن پنل مدیریت وب به سرور موجود
sudo tuwunel enable-elementx  # فعال‌سازی ثبت‌نام خودکار Element X
sudo tuwunel autobackup       # فعال‌سازی پشتیبان‌گیری رمزنگاری‌شدهٔ هفتگی خودکار
sudo tuwunel privacy          # حریم خصوصی/فراداده: ببینید چه‌چیزی حذف‌شدنی است، لاگ‌ها را پاک کنید
sudo tuwunel forget-secrets   # ضدجرم‌یابی: ماسک‌کردن رمز/کد دعوت متن‌ساده روی دیسک
sudo tuwunel uninstall        # حذف
```

> 🔄 **پیکربندی مجدد بدون نصب دوباره** (منوی ۳): گزینه‌ها را دوباره می‌پرسد، **Enter = حفظ مقدار فعلی**، سپس راه‌اندازی مجدد. حساب‌ها، تاریخچه و کلیدها همه حفظ می‌شوند.

## 🔒 حریم خصوصی / ضدجرم‌یابی (تمرکز این نسخه)

این سامانه برای حفاظت از ارتباطات محرمانه طراحی شده و **به‌صورت پیش‌فرض ردپای باقی‌مانده روی سرور را کمینه می‌کند**:

- ✅ **محتوای پیام‌ها با رمزنگاری سرتاسری** — نه شما (اپراتور) و نه شرکت میزبان نمی‌توانید آن را بخوانید. این تنها تضمینی است که مستقل از خوب یا بد بودن سرور برقرار می‌ماند.
- ✅ **کمینه‌سازی فراداده (پیش‌فرض روشن):** IP واقعی کاربر ذخیره نمی‌شود (`ip_source`)، **حذف‌ها واقعاً دائمی‌اند** (نسخهٔ اصل ۶۰ روز نگه داشته نمی‌شود)، حضور خاموش، سطح لاگ آن‌قدر پایین که IP ثبت نمی‌شود، محدودسازی نمایش پروفایل/فهرست اتاق‌ها.
- ✅ **ابزارهای ضدجرم‌یابی:** `sudo tuwunel forget-secrets` رمز/کد دعوت متن‌ساده روی دیسک را ماسک می‌کند؛ `sudo tuwunel autobackup` پشتیبان‌ها را با AES-256 رمزنگاری می‌کند.

**مرزهای صادقانه (همین‌ها را به مشتری بگویید؛ بیش از حد وعده ندهید):**

- ❌ **فراداده‌ای که حذف‌شدنی نیست:** عضویت اتاق‌ها، خط زمانی رویدادها، وجود حساب، نام اتاق‌ها / نام فایل‌ها — سرور برای کارکرد باید آن‌ها را نگه دارد. E2E *محتوا* را حفظ می‌کند، **نه فراداده را** (چه کسی با چه کسی، چه زمانی، نام اتاق، نام فایل).
- ❌ **اگر دیسک به‌طور فیزیکی توقیف/تصویربرداری شود:** روی یک VPS معمولی داده‌ها متن‌ساده روی دیسک نوشته می‌شوند و فراداده با جرم‌یابی قابل استخراج است. دفاع از این لایه به ارائه‌دهنده/حوزهٔ قضایی مورد اعتماد، یا رمزنگاری کامل دیسک پیشرفته / پروکسی «سپر» جلویی نیاز دارد (خودکار نیست؛ کار مهندسی می‌خواهد).
- ❌ **پنهان‌کردن IP سرور با Cloudflare عملاً کار نمی‌کند** (لاگ‌های شفافیت گواهی و زیردامنه‌های ابر خاکستری آن را لو می‌دهند). پنهان‌کردن واقعی IP نیازمند یک پروکسی WireGuard جلویی خودمیزبان است.

> عبارت دقیق برای مشتریان: **«محتوای پیام‌ها و پیوست‌ها با رمزنگاری سرتاسری محافظت می‌شوند؛ اپراتور نمی‌تواند آن‌ها را بخواند. اما سرور فرادادهٔ ارتباط (چه کسی در کدام اتاق، چه زمانی، نام فایل‌ها، نام اتاق‌ها) را نگه می‌دارد که در صورت توقیف فیزیکی سرور قابل استخراج است.»**

## 💾 پشتیبان‌گیری (به‌شدت توصیه می‌شود)

اگر پایگاه‌داده یا کلیدها را از دست بدهید **بازیابی ممکن نیست**. این نسخه PostgreSQL ندارد: پشتیبان صرفاً بایگانی `data/tuwunel` (پایگاه‌داده + رسانه) + `tuwunel.toml` + `.env` است.

**توصیه: پشتیبان‌گیری رمزنگاری‌شدهٔ خودکار را روشن کنید** (هفتگی، AES-256، با چرخش خودکار، در صورت پُر بودن دیسک به‌طور خودکار رد می‌شود):

```bash
sudo tuwunel autobackup     # پوشه / تعداد نگهداری / بازه را تنظیم کنید؛ کلید نمایش داده می‌شود — حتماً در مدیر رمز ذخیره کنید
```

**یا اکنون پشتیبان بگیرید** (منوی ۵؛ می‌توانید عبارت عبور رمزنگاری تعیین کنید).

**دانلود به رایانهٔ خود** (روی دستگاه خودتان اجرا کنید):

```bash
scp root@IP_سرور_شما:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **کلید پشتیبان را ذخیره کنید.** بدون آن، پشتیبان رمزنگاری‌شده هرگز باز نمی‌شود. کلید فقط روی سرور است؛ اگر سرور از بین برود و کلید را ذخیره نکرده باشید، پشتیبان‌ها بی‌فایده‌اند.
> 💡 پوشهٔ محلی `backups/` همراه سرور از بین می‌رود. فایل‌های `.enc` را مرتب جای دیگری کپی کنید، یا پوشهٔ پشتیبان را روی یک حجم خارجیِ متصل / ذخیره‌سازی شیءگرا قرار دهید.

## ❓ پرسش‌های پرتکرار

| مشکل | علت و راه‌حل |
|---|---|
| **Element X گوشی متصل نمی‌شود / «couldn't connect to this homeserver»** | **ابتدا تاریخ و ساعت گوشی را بررسی کنید** (تنظیمات → تاریخ و ساعت → «خودکار»): ساعت اشتباه اعتبارسنجی گواهی را شکست می‌دهد؛ رایج‌ترین علت (به‌ویژه Android). سپس: شبکهٔ دیگری را امتحان کنید، مطمئن شوید در مرورگر گوشی `https://دامنهٔ‌شما/.well-known/matrix/client` باز می‌شود، و Element X را به‌روز کنید. |
| **Element X می‌گوید «برای پشتیبانی از سرویس احراز هویت باید ارتقا یابد»** | مربوط به ثبت‌نام است. مطمئن شوید ثبت‌نام موبایل روشن است (`sudo tuwunel enable-elementx`) و tuwunel به‌روز است (`docker compose pull tuwunel`). یا از طریق وب / `adduser` ثبت‌نام کنید و اعضا با **ورود رمزی** در Element X وارد شوند. |
| **فایل‌های بزرگ آپلود نمی‌شوند** | سقف اندازه را بررسی کنید (`MAX_UPLOAD=10G sudo -E tuwunel config`)؛ با Cloudflare، میزبان **`matrix` باید ابر خاکستری باشد** (سقف ۱۰۰ مگابایتیِ ابر نارنجی فایل‌های بزرگ را می‌کشد). |
| **پنل admin.دامنه باز نمی‌شود / خطای ورود** | معمولاً رکورد DNS `admin` نیست یا گواهی هنوز آماده نیست — رکورد را اضافه و چند دقیقه صبر کنید. برای خطای ورود: `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`. |
| رمز مدیر را فراموش کردم | `cat /opt/tuwunel/CREDENTIALS.txt`؛ یا `sudo tuwunel adduser` برای ساخت مدیر جدید. |
| تغییر پیکربندی (تماس، ثبت‌نام، اندازهٔ فایل، مدیریت) | بدون نصب دوباره: `sudo tuwunel config` — Enter مقادیر را حفظ می‌کند، داده/حساب‌ها دست‌نخورده. |
| تماس وصل می‌شود اما صدا/تصویر نیست | ۹۹٪ مواقع پورت **7882/UDP** اجازه ندارد — در فایروال ارائه‌دهنده اضافه کنید. |
| دیسک پُر می‌شود | منوی ۷ «پاک‌سازی دیسک»؛ برای مصرف سنگین یک حجم بزرگ یا ذخیره‌سازی شیءگرا تدارک ببینید. |
| «قابل رمزگشایی نیست» برای پیام‌های قدیمی در اتاق رمزنگاری‌شده | طبیعی است: دستگاه جدید کلیدهای تاریخی ندارد. نشست جدید را از یک دستگاه قدیمی تأیید کنید. |
| حذف / نصب مجدد | `sudo tuwunel uninstall` (تأیید دوگانه پیش از حذف)؛ نصب مجدد = حذف و اجرای دوبارهٔ فرمان نصب. |

## 📦 مؤلفه‌های سمت سرور

پس از نصب، این مؤلفه‌ها اجرا می‌شوند، همه با Docker هماهنگ می‌شوند، متن‌باز و قابل ممیزی:

```
Caddy (HTTPS خودکار) + tuwunel (Rust، RocksDB داخلی، بدون PostgreSQL)
  + Element Web (کلاینت وب خودتان، اختیاری)
  + Ketesa (پنل مدیریت گرافیکی، اختیاری)
  + LiveKit + lk-jwt-service (تماس، اختیاری)
```

مسیر نصب `/opt/tuwunel`. تمام منطق در یک اسکریپت واحد `matrix-tuwunel-installer.sh` است که خودتان می‌توانید ممیزی کنید.
