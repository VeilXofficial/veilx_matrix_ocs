# プライベート Matrix 通信サーバー · ワンコマンド展開（tuwunel エディション）

**あなたのサーバー、あなたのデータ — 機密性とデータ主権のために作られた、セルフホスト型チームメッセンジャー。**

**tuwunel** エンジン（Rust製、データベース内蔵、**PostgreSQL 不要**）採用で、より軽く・安定し、**Telegram のように大きなファイル／写真／長い動画を送信**できます。RAM 2GB で中規模チームを余裕をもって運用。エンドツーエンド暗号化、メタデータ最小化、招待制登録が**デフォルトで有効**。コマンド1行で展開 — システム管理者でなくても大丈夫、各オプションにわかりやすい説明が付いています。

推奨クライアント：**Element X**。専用の **VeilX** クライアントを開発中（より洗練され使いやすく機能も豊富、オープンソースで監査可能。運用チームは英国・シンガポール・日本などに拠点）。顧客資料、契約書、社内の議論、音声・ビデオ会議 — すべてがあなた自身のサーバーだけに存在します。オープンプロトコル [Matrix](https://matrix.org) がベース。ソースは公開・監査可能（非商用は無料）。

## ✨ 導入すると何ができる

- 💬 テキストチャットとグループルーム（**エンドツーエンド暗号化 — サーバーもホスティング業者もメッセージ内容を見られません**）
- 📁 **大きなファイル／大きな写真／長い動画を送信**（1ファイルの上限はデフォルト **4GB**、さらに大きく設定可能 — 目玉機能）
- 📞 1対1・グループの音声／ビデオ通話（任意）
- 📱 スマホからセルフ登録：**Element X** をインストール → ドメインを入力 → 登録・ログイン（element.io に行く必要なし）
- 🌐 自前ドメインの Web クライアント：ブラウザで `https://あなたのドメイン` を開いて登録／ログイン、アプリ不要
- 🖥️ **グラフィカルな Web 管理コンソール**（Ketesa）：ユーザー管理、招待コードの発行／取り消し、部屋やメディアの確認
- 🔒 **デフォルトでプライバシー強化**：実クライアント IP を保存しない、削除（取り消し）は永久、プレゼンス無効、ログに IP を記録しない
- 👥 招待制：デフォルトでは招待コードを持つ人だけが登録可能 — 外部者は入れません
- ⚡ **省リソース**：Rust の単一プロセス、PostgreSQL 不要 — RAM 2GB で約300人

| 何が | 要件 | どこで |
|---|---|---|
| **クラウドサーバー（VPS）** | Ubuntu 22.04 / 24.04（または Debian 11+）、グローバル IP。**チーム規模に合わせて RAM を選択**（下記参照）。**大きなファイルを共有するなら十分なディスクを確保。** | [Vultr](https://www.vultr.com)、[DigitalOcean](https://www.digitalocean.com)、[Hetzner](https://www.hetzner.com/cloud)、[さくらのクラウド](https://cloud.sakura.ad.jp)、[ConoHa](https://www.conoha.jp) など信頼できる業者 |
| **ドメイン** | どの TLD でも可（.com / .org / .net…） | [Namecheap](https://www.namecheap.com)、[Porkbun](https://porkbun.com)、[お名前.com](https://www.onamae.com) など |
| **10分** | すべてコピー＆ペースト、プログラミング不要 | — |

> 💡 **同時アクティブ人数で RAM を選ぶ**（連合オフ、チャット＋ファイル）：**1GB** ≈ 数十人（通話はオフに）· **2GB** ≈ 300人 · **4GB / 2 vCPU** ≈ 500人。tuwunel は軽量で、CPU がボトルネックになることは稀です。
> 💡 **大きなファイルではディスクが本当の変数です。** メディアはサーバーのローカルディスクに保存され、エンドツーエンド暗号化のため重複排除ができません。ヘビーユースでは月に数百 GB〜TB 増えることも。システムディスクは ≥ 50〜100GB を推奨。ヘビーユーザーは大容量ボリュームやオブジェクトストレージを別途用意し、メニューの「ディスク整理」に注意してください。
> 💡 **プライバシーには管轄地が重要です。** 信頼できる業者と国を選んでください — ホストは原則として協力を強制され得ます。E2E 暗号化は*内容*を保護しますが、メタデータはホスト側に残ります。
> 💡 注文時は必ず **Ubuntu 22.04 / 24.04** イメージを選択。RAM が 2.5GB 未満の場合、スクリプトが自動でスワップを追加します。

## ステップ0 — OS の準備（正しく注文済みなら省略）

対応は **Ubuntu 22.04 / 24.04**（または Debian 11+）のみ。

- **新規サーバー**：注文時に「OS / Image」欄で **Ubuntu 22.04 x86_64**（または 24.04）を選択。
- **OS が違う**：業者の「再インストール / Reinstall / Rebuild」で Ubuntu 22.04 を導入。⚠️ これは**全データを消去**します。既存サーバーは先にバックアップを。
- 今の OS が不明？接続後（ステップ3）に `cat /etc/os-release` を実行。
- 間違った OS に入れても、スクリプトが検知して明確に知らせます。何も壊しません。

## ステップ1 — DNS レコードを追加（約2分）

**ドメイン登録業者**の DNS パネルで **A レコード**を追加します。

ドメイン `mychat.org`、サーバー IP `1.2.3.4` の場合：

| 種別 | ホスト（名前） | 値 | 用途 |
|---|---|---|---|
| A | `@`（ルートドメイン） | `1.2.3.4` | ルートドメイン + Web クライアント + 委任 |
| A | `matrix` | `1.2.3.4` | ホームサーバー |
| A | `admin` | `1.2.3.4` | Web 管理コンソール（管理オフなら省略） |
| A | `livekit` | `1.2.3.4` | 通話（通話オフなら省略） |
| A | `matrix-rtc` | `1.2.3.4` | 通話（通話オフなら省略） |

> 📌 ホスト欄には**接頭辞**（`matrix`）だけを入れ、完全なドメインは入れないこと。
> 📌 レコードは1〜10分で伝播します。未反映でもインストーラーが自動で待機します。
> 📌 チャット＋ファイルのみ（通話オフ）なら `@` / `matrix` / `admin` の3つで十分。
> ⚠️ **Cloudflare を使う場合**：これらのレコードは必ず**グレークラウド（DNS のみ）**に。**オレンジクラウド（プロキシ）を有効にしない**でください。プロキシは ① 大きなファイルを 100MB に制限（Free/Pro）② 証明書の発行を妨げる ③ 通話メディアを遮断 します。インストール時に CDN 使用の有無を聞かれます（使うなら「はい」で DNS チェックを緩和）が、`matrix` ホストはグレークラウドのままに。

## ステップ2 — ファイアウォールのポートを開放（約1分）

業者の「セキュリティグループ / ファイアウォール」で許可：

| ポート | プロトコル | 用途 |
|---|---|---|
| 80 | TCP | HTTPS 証明書の発行 |
| 443 | TCP + UDP | Web と暗号化通信 |
| 7881 | TCP | 通話の予備チャネル（通話オフなら省略） |
| **7882** | **UDP** | **通話の音声・映像（最も忘れやすい！ 抜けると音も映像も出ません）** |

> 📌 業者に「セキュリティグループ」の層が無ければスキップ — スクリプトがシステムのファイアウォールを自動設定します。

## ステップ3 — サーバーへ接続（約1分）

ターミナルを開き（macOS：「ターミナル」、Windows：「PowerShell」）、次を実行（IP を置換）し、求められたらサーバーのパスワードを入力してください（**パスワード入力中に画面に何も表示されないのは正常です**）：

```bash

```

初回接続でフィンガープリントの確認が出たら `yes` と入力。

> 📌 デフォルトユーザーが root でない場合（例：`ubuntu`）は `ssh ubuntu@IP`。スクリプトが自動で権限昇格します。

## ステップ4 — インストーラーを実行（約5〜10分、自動）

接続後、**1行まるごとコピー**してターミナルに貼り付け：

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

ウィザードに従ってください。**6つのオプション + CDN の質問**があり、各項目は画面に説明が出るので、迷ったら Enter を（デフォルトが最も安全な組み合わせ）：

| オプション | Enter（推奨） | 説明 |
|---|---|---|
| 1 誰が登録できるか | **招待コード必須** | コードを持つ人だけ登録可、最初に登録した人が管理者。（完全開放も可だが業務用途は非推奨） |
| 2 連合（Federation） | **オフ** | 島モード：外部から自分のメンバーへメッセージ不可、攻撃面が最小。 |
| 3 音声・ビデオ通話 | **まずオフ** | DNS を2件（`livekit`/`matrix-rtc`）とポート 7881/7882 が必要。まずチャット＋ファイルを安定させる。 |
| 4 Web クライアント | **オン** | メンバーが `https://あなたのドメイン` で登録・ログイン、アプリ不要。 |
| 5 Web 管理コンソール | **オン** | `admin.あなたのドメイン` にグラフィカルなコンソール Ketesa。 |
| 6 最大ファイルサイズ | **4G** | 好きな値（例：10G）に。大きいほどディスクを消費。 |
| ＋ Cloudflare/CDN の背後か | **いいえ** | CDN を使う場合のみ「はい」（DNS チェックを緩和）。`matrix` はグレークラウドのまま。 |

> 🔒 **自動で有効（質問なし、インストール後すぐ適用）：** ① 新しい部屋は **E2E 暗号化を強制** ② **メタデータ最小化**（実 IP 非保存、削除は永久、プレゼンス無効、ログに IP 非記録） ③ **Element X スマホ・セルフ登録**（ネイティブ OIDC、登録には引き続き招待コードが必要）。メタデータ強化を切るには：`PRIVACY=0 sudo -E tuwunel config`。
> 🤖 **完全無人インストール：** `REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

以降は自動：DNS 確認 → Docker 導入 → システム調整 → ファイアウォール → 設定生成 → サービス起動 → HTTPS 証明書取得 → **管理者を自動作成しアカウント／パスワードを表示**。

成功はこう表示されます：

```
========================================================
 🎉 tuwunel 展開完了!  mychat.org
 (登録[token] · 連合[off] · 通話[off] · Web[on] · 管理[on] · モバイル登録[on] · 大ファイル[4G] · エンジン tuwunel/Rust, Postgres なし)

 メンバーの登録 / ログイン
   Web(推奨): https://mychat.org を開いて登録・ログイン
   モバイルアプリ: Element X をインストール → サーバー "mychat.org" → 登録(招待コード)またはログイン

 管理者(自動作成済み)
   アカウント: admin    パスワード: xxxxxxxxxxxx
   Web 管理コンソール: https://admin.mychat.org  (上の管理者アカウントでログイン)

 日常管理:  sudo tuwunel   (メニュー)    sudo tuwunel adduser   (メンバー追加)
========================================================
```

> 🔑 **管理者アカウント／パスワードと招待コードを控えてください！** すべてサーバーの `/opt/tuwunel/CREDENTIALS.txt` にあります（`cat /opt/tuwunel/CREDENTIALS.txt`）。
> ⚠️ 最後に「まだ準備できていません」と出たら、たいていはクラウドのファイアウォールが 80/443 を許可していないか、DNS がまだ世界的に伝播していないためです。Caddy が証明書を自動で再試行するので、再インストールは不要。

## 👥 メンバーの参加方法（いずれか1つ）

1. **管理者がアカウント作成（最も統制的）：** `sudo tuwunel adduser` でユーザー作成とパスワード設定を一度に。「ドメイン + ユーザー名 + パスワード」を渡します。招待コードは外に出ません。
2. **招待コードを配布、Web セルフ登録：** コード（`CREDENTIALS.txt` 内）+ `https://あなたのドメイン` を渡すと、メンバーがブラウザで自分で登録。
3. **招待コードを配布、Element X セルフ登録：** Element X をインストール → ドメイン入力 → アカウント作成 → 招待コード入力。

## 🖥️ グラフィカル管理コンソール（Ketesa、推奨）

インストール時（オプション5オン）に**グラフィカルな Web 管理コンソール**が自動展開されます。コマンド不要、すべてマウスで管理。

**入り方：** `https://admin.あなたのドメイン` を開く → **管理者アカウント `admin` + パスワードで直接ログイン**（別のゲートパスワードは不要、コンソールはあなたのサーバーに固定）。

**できること：** ユーザー管理（作成／無効化／パスワード再設定）、**招待コードの発行と取り消し**、部屋・メディアの確認、部屋ディレクトリ、スケジュールタスク。

> 📌 「通報イベント／通報されたユーザー」ページはエラーや空表示になることがあります — tuwunel がその機能を実装していないためで、**正常**です。ほぼ使いません。
> 📌 コンソールには tuwunel v1.8.1+ が必要（`:latest` イメージに含む）。コンソールのログインでエラーが出たら `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`。
> 📌 既存サーバーにコンソールを追加：`sudo tuwunel enable-admin`（先に `admin` の DNS レコードを追加）。

```
┌──────────────────────────────────────────────┐
│  tuwunel 管理メニュー   あなたのドメイン       │
└──────────────────────────────────────────────┘
  1) ステータス表示          6) サービスイメージを更新
  2) メンバー追加            7) ディスク整理
  3) 設定変更                8) 全サービス再起動
  4) Web 管理コンソール 切替  9) スクリプト更新 + 新機能適用
  5) 今すぐバックアップ      10) 完全アンインストール
```

コマンドを直接使うことも：

```bash
sudo tuwunel adduser          # メンバー追加
sudo tuwunel config           # 設定変更(Enter=維持、データ/アカウントはそのまま)
sudo tuwunel update           # GitHub から最新スクリプトを取得し新機能を適用(データはそのまま)

sudo tuwunel uninstall        # アンインストール
```

## 🔒 プライバシー / アンチフォレンジック（本エディションの重点）

本システムは機密通信の保護のために設計され、**デフォルトでサーバーに残る痕跡を最小化**します：

- ✅ **メッセージ内容はエンドツーエンド暗号化** — あなた（運用者）もホスティング業者も読めません。サーバーの良し悪しに関係しない唯一の保証です。
- ✅ **メタデータ最小化（デフォルトオン）：** 実クライアント IP を非保存（`ip_source`）、**削除（取り消し）は本当に永久**（原文を60日保持しない）、プレゼンス無効、ログレベルが低く IP を記録しない、プロフィール／部屋ディレクトリの露出を厳格化。
- ✅ **アンチフォレンジック・ツール：** `sudo tuwunel forget-secrets` はディスク上の平文パスワード／招待コードを墨消し。`sudo tuwunel autobackup` はバックアップを AES-256 で暗号化。

**正直な限界（クライアントにそのまま説明を。過大な約束はしない）：**

- ❌ **削除できないメタデータ：** 部屋のメンバー関係、イベントのタイムライン、アカウントの存在、部屋名／ファイル名 — サーバーが動作するために保持が必須です。E2E は*内容*を守りますが、**メタデータ（誰と誰が、いつ、部屋名、ファイル名）は守りません**。
- ❌ **ディスクが物理的に押収／複製された場合：** 一般的な VPS ではデータは平文でディスクに書かれ、フォレンジックでメタデータを抽出できます。この層の防御には、信頼できる業者／管轄地、または高度なフルディスク暗号化 / 前段「シールド」プロキシ（自動ではなく、エンジニアリングが必要）が要ります。
- ❌ **Cloudflare でサーバー IP を隠すのは実質的に無効**（証明書透明性ログやグレークラウドのサブドメインが漏らします）。本当に IP を隠すには、自前の WireGuard 前段プロキシが必要です。

> クライアント向けの正確な文言：**「メッセージ内容と添付ファイルはエンドツーエンド暗号化されており、運用者も読めません。ただしサーバーは通信のメタデータ（誰がどの部屋に、いつ、ファイル名、部屋名）を保持し、サーバーが物理的に押収された場合に抽出され得ます。」**

## 💾 バックアップ（強く推奨）

データベースや鍵を失うと**復元できません**。本エディションに PostgreSQL は無く、バックアップは `data/tuwunel`（DB + メディア）+ `tuwunel.toml` + `.env` をまとめるだけです。

**推奨：自動暗号化バックアップを有効化**（週次、AES-256、自動ローテーション、ディスクがほぼ満杯なら自動スキップ）：

```bash
sudo tuwunel autobackup     # フォルダ/保持数/頻度を設定。鍵が表示されるので必ずパスワードマネージャーへ
```

**または今すぐバックアップ**（メニュー5、暗号化パスフレーズを設定可能）。

**PC へダウンロード**（自分のマシンで実行）：

```bash
scp root@あなたのサーバーIP:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **バックアップ鍵を必ず控えること。** 鍵が無ければ暗号化バックアップは永遠に開けません。鍵はサーバーにのみ存在するため、サーバーが失われ鍵も控えていなければバックアップは無意味です。
> 💡 ローカルの `backups/` フォルダはサーバーと共に消えます。`.enc` を定期的に別の場所へコピーするか、バックアップフォルダをマウント済み外部ボリューム／オブジェクトストレージに指定してください。

## ❓ よくある質問

| 問題 | 原因と対処 |
|---|---|
| **スマホの Element X が接続できない／「couldn't connect to this homeserver」** | **まずスマホの日付と時刻を確認**（設定→日付と時刻→「自動」）：時計がずれていると証明書検証に失敗、最も多い原因（特に Android）。次に：別のネットワークを試す、スマホのブラウザで `https://あなたのドメイン/.well-known/matrix/client` が開くか確認、Element X を最新に更新。 |
| **Element X が「認証サービス対応のため更新が必要」と表示** | 登録に関するものです。モバイル登録が有効か（`sudo tuwunel enable-elementx`）と tuwunel が最新か（`docker compose pull tuwunel`）を確認。または Web / `adduser` で登録し、メンバーは Element X で**パスワードログイン**。 |
| **大きなファイルがアップロードできない** | サイズ上限を確認（`MAX_UPLOAD=10G sudo -E tuwunel config`）。Cloudflare 利用時は **`matrix` ホストをグレークラウドに**（オレンジクラウドの 100MB 上限が大ファイルを潰します）。 |
| **管理コンソール admin.ドメイン が開かない／ログインエラー** | たいてい `admin` の DNS レコードが無いか証明書が未発行。レコードを追加し数分待つ。ログインエラーは `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`。 |
| 管理者パスワードを忘れた | `cat /opt/tuwunel/CREDENTIALS.txt`；または `sudo tuwunel adduser` で新しい管理者を作成。 |
| 設定変更（通話、登録、ファイルサイズ、管理） | 再インストール不要：`sudo tuwunel config` — Enter で値を維持、データ/アカウントはそのまま。 |
| 通話は繋がるが音・映像が出ない | 99% はポート **7882/UDP** が未許可。業者のファイアウォールで追加。 |
| ディスクが一杯 | メニュー7「ディスク整理」；ヘビーユーザーは大容量ボリュームやオブジェクトストレージを用意。 |
| 暗号化部屋の古いメッセージが「復号できない」 | 正常：新しい端末に過去の鍵がありません。古い端末から新しいセッションを検証してください。 |
| アンインストール / 再インストール | `sudo tuwunel uninstall`（二重確認後に削除）；再インストール = アンインストール後にインストールコマンドを再実行。 |

## 📦 サーバー側コンポーネント

インストール後、次のコンポーネントが動作します。すべて Docker でオーケストレーションされ、オープンソースで監査可能：

```
Caddy(自動 HTTPS) + tuwunel(Rust, RocksDB 内蔵, PostgreSQL なし)
  + Element Web(自前 Web クライアント, 任意)
  + Ketesa(グラフィカル管理コンソール, 任意)
  + LiveKit + lk-jwt-service(通話, 任意)
```

インストール先 `/opt/tuwunel`。すべてのロジックは単一スクリプト `matrix-tuwunel-installer.sh` にあり、自分で監査できます。

## 🆚 tuwunel エディション vs Synapse

リポジトリには Synapse エディション（`matrix-installer.sh`）もあります。要約：

- **tuwunel エディション（本ドキュメント）：** Rust、PostgreSQL 不要、**より効率的で大ファイルに強い**、2GB で約300人。ほとんどのチーム、特に大ファイルを多用する場面に最適。
- **Synapse エディション：** Python + PostgreSQL、エコシステムと管理ツールが最も成熟しているが、RAM を多く使い大ファイルには弱い。

**新規展開では tuwunel エディションを推奨。**
