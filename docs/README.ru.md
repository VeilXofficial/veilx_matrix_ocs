# Приватный сервер Matrix · Развёртывание одной командой (редакция tuwunel)

**Ваш сервер, ваши данные — самостоятельно размещаемый командный мессенджер, созданный ради конфиденциальности и суверенитета данных.**

На движке **tuwunel** (Rust, встроенная база данных, **без PostgreSQL**): легче, стабильнее и умеет **пересылать большие файлы / фото / длинные видео, как Telegram**. 2 ГБ ОЗУ комфортно обслуживают средний коллектив. Сквозное шифрование, минимизация метаданных и регистрация только по приглашению **включены по умолчанию**. Развёртывание одной командой — не нужно быть системным администратором; каждый пункт снабжён понятным пояснением.

Рекомендуемый клиент: **Element X**. Собственный клиент **VeilX** в разработке (аккуратнее, удобнее, больше функций; открытый код, поддаётся аудиту; команда эксплуатации в Великобритании, Сингапуре и Японии). Файлы клиентов, договоры, внутренние обсуждения, голосовые и видеовстречи — всё это существует только на вашем сервере. Построено на открытом протоколе [Matrix](https://matrix.org). Исходный код открыт и доступен для аудита (бесплатно для некоммерческого использования).

## ✨ Что вы получаете после установки

- 💬 Текстовый чат и групповые комнаты (**сквозное шифрование — ни сервер, ни хостинг-провайдер не видят содержимое сообщений**)
- 📁 **Отправка больших файлов / фото / длинных видео** (лимит на файл по умолчанию **4 ГБ**, настраивается больше — ключевая функция)
- 📞 Личные и групповые голосовые / видеозвонки (опционально)
- 📱 Регистрация со смартфона: установите **Element X**, введите свой домен, зарегистрируйтесь и войдите (без захода на element.io)
- 🌐 Собственный веб-клиент: откройте `https://ваш-домен` в браузере для регистрации/входа, без приложения
- 🖥️ **Графическая веб-панель администратора** (Ketesa): управление пользователями, выпуск/отзыв кодов приглашений, просмотр комнат и медиа
- 🔒 **Усиление приватности по умолчанию**: реальный IP клиента не сохраняется, удаления необратимы, статус присутствия отключён, логи не пишут IP
- 👥 Только по приглашению: по умолчанию зарегистрироваться может лишь тот, у кого есть код приглашения
- ⚡ **Экономно к ресурсам**: один процесс на Rust, без PostgreSQL — 2 ГБ ОЗУ на ~300 человек

## 📋 Перед началом (3 вещи)

| Что | Требование | Где |
|---|---|---|
| **Облачный сервер (VPS)** | Ubuntu 22.04 / 24.04 (или Debian 11+), публичный IP. **Выбирайте ОЗУ по размеру команды** (см. ниже). **Выделите достаточно диска, если делитесь большими файлами.** | [Vultr](https://www.vultr.com), [DigitalOcean](https://www.digitalocean.com), [Hetzner](https://www.hetzner.com/cloud), [OVHcloud](https://www.ovhcloud.com) или другой провайдер, которому вы доверяете |
| **Домен** | Любая зона (.com / .org / .net…) | [Namecheap](https://www.namecheap.com), [Porkbun](https://porkbun.com), [Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) |
| **10 минут** | Всё через копирование-вставку; программировать не нужно | — |

> 💡 **Выбирайте ОЗУ по числу одновременно активных пользователей** (федерация выключена, чат + файлы): **1 ГБ** ≈ несколько десятков человек (выключите звонки) · **2 ГБ** ≈ 300 человек · **4 ГБ / 2 vCPU** ≈ 500 человек. tuwunel лёгкий, CPU редко становится узким местом.
> 💡 **Диск — реальная переменная для больших файлов.** Медиа хранится на локальном диске сервера и, будучи сквозно зашифрованным, не поддаётся дедупликации. При активном использовании может прибавляться от сотен ГБ до ТБ в месяц. Системный диск — ≥ 50–100 ГБ; при активном использовании выделите большой том или объектное хранилище и следите за пунктом меню «Очистить диск».
> 💡 **Для приватности важна юрисдикция.** Выбирайте провайдера и страну, которым доверяете — хост в принципе можно принудить к сотрудничеству. Сквозное шифрование в любом случае защищает *содержимое*, но метаданные остаются на хосте.
> 💡 При заказе всегда выбирайте образ **Ubuntu 22.04 / 24.04**. Если ОЗУ меньше 2,5 ГБ, скрипт автоматически добавит swap.

## Шаг 0 — Подготовка ОС (пропустите, если заказали правильно)

Поддерживается только **Ubuntu 22.04 / 24.04** (или Debian 11+).

- **Новый сервер**: выберите **Ubuntu 22.04 x86_64** (или 24.04) в поле «ОС / Image» при заказе.
- **Неверная ОС**: используйте «Переустановить / Reinstall / Rebuild» у провайдера и поставьте Ubuntu 22.04. ⚠️ Это **сотрёт все данные** — сначала сделайте резервную копию существующего сервера.
- Не знаете, что запущено? После подключения (Шаг 3) выполните `cat /etc/os-release`.
- Если установить на неверную ОС, скрипт это обнаружит и чётко сообщит; ничего не сломается.

## Шаг 1 — Добавьте DNS-записи (~2 минуты)

В DNS-панели вашего **регистратора домена** добавьте **записи A**.

Пусть домен `mychat.org`, IP сервера `1.2.3.4`:

| Тип | Хост (имя) | Значение | Назначение |
|---|---|---|---|
| A | `@` (корневой домен) | `1.2.3.4` | Корневой домен + веб-клиент + делегирование |
| A | `matrix` | `1.2.3.4` | Домашний сервер |
| A | `admin` | `1.2.3.4` | Веб-панель администратора (пропустить, если админ выкл.) |
| A | `livekit` | `1.2.3.4` | Звонки (пропустить, если звонки выкл.) |
| A | `matrix-rtc` | `1.2.3.4` | Звонки (пропустить, если звонки выкл.) |

> 📌 В поле хоста — только **префикс** (`matrix`), а не полный домен.
> 📌 Записи распространяются за 1–10 минут; установщик сам подождёт, если они ещё не активны.
> 📌 Для чата + файлов (без звонков) нужны только `@` / `matrix` / `admin`.
> ⚠️ **Используете Cloudflare?** Эти записи должны быть в режиме **серого облака (только DNS)**; **не включайте оранжевое облако (прокси)**. Прокси ① ограничит большие файлы 100 МБ (Free/Pro) ② помешает выпуску сертификата ③ заблокирует медиа звонков. Установщик спросит, используете ли вы CDN (ответьте «да», чтобы ослабить проверку DNS), но хост `matrix` должен оставаться в сером облаке.

## Шаг 2 — Откройте порты фаервола (~1 минута)

В «Security Group / Firewall» вашего провайдера разрешите:

| Порт | Протокол | Назначение |
|---|---|---|
| 80 | TCP | Выпуск HTTPS-сертификата |
| 443 | TCP + UDP | Веб и зашифрованный трафик |
| 7881 | TCP | Резервный канал звонков (пропустить без звонков) |
| **7882** | **UDP** | **Аудио/видео звонков (легче всего забыть! Без него — нет звука и картинки)** |

> 📌 Если у провайдера нет слоя «security group» — пропустите: скрипт сам настроит системный фаервол.

## Шаг 3 — Подключитесь к серверу (~1 минута)

Откройте терминал (macOS: «Терминал»; Windows: «PowerShell»), выполните следующее (замените IP) и введите пароль сервера, когда его запросят (**то, что при вводе пароля на экране ничего не отображается — это нормально**):

```bash
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

При первом подключении ответьте `yes` на запрос об отпечатке.

> 📌 Если пользователь по умолчанию не root (например, `ubuntu`) — используйте `ssh ubuntu@IP`; скрипт сам повысит права.

## Шаг 4 — Запустите установщик (~5–10 минут, автоматически)

После подключения **скопируйте всю строку** и вставьте в терминал:

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

Следуйте мастеру. Он задаёт **6 вопросов + вопрос о CDN**; каждый поясняется на экране, поэтому при сомнении просто нажимайте Enter (значения по умолчанию — самая безопасная комбинация):

| Опция | Enter (рекомендуется) | Примечания |
|---|---|---|
| 1 Кто может регистрироваться | **Требуется код приглашения** | Зайдёт только тот, у кого есть ваш код; первый зарегистрировавшийся становится админом. (Или полностью открыто — не для бизнеса.) |
| 2 Федерация | **Выкл.** | Режим острова: снаружи никто не напишет вашим участникам; минимальная поверхность атаки. |
| 3 Голосовые/видеозвонки | **Сначала выкл.** | Нужны две доп. DNS-записи (`livekit`/`matrix-rtc`) и порты 7881/7882; сначала стабилизируйте чат + файлы. |
| 4 Веб-клиент | **Вкл.** | Участники открывают `https://ваш-домен` для регистрации/входа, без приложения. |
| 5 Панель администратора | **Вкл.** | Графическая панель (Ketesa) на `admin.ваш-домен`. |
| 6 Максимальный размер файла | **4 ГБ** | Ставьте любое (например, 10 ГБ); больше размер — больше диска. |
| ＋ За Cloudflare/CDN? | **Нет** | «Да» только если используете CDN (ослабляет проверку DNS); `matrix` всё равно в сером облаке. |

> 🔒 **Включается автоматически (не спрашивается; действует сразу после установки):** ① новые комнаты **принудительно с E2E-шифрованием** ② **минимизация метаданных** (реальный IP не сохраняется, удаления необратимы, присутствие выкл., логи без IP) ③ **саморегистрация через Element X** (нативный OIDC; регистрация по-прежнему требует кода приглашения). Отключить усиление метаданных: `PRIVACY=0 sudo -E tuwunel config`.
> 🤖 **Полностью без участия:** `REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

Далее всё автоматически: проверка DNS → установка Docker → тюнинг системы → фаервол → генерация конфигурации → запуск сервисов → выпуск HTTPS-сертификата → **автосоздание админа и вывод логина/пароля**.

Успех выглядит так:

```
========================================================
 🎉 tuwunel развёрнут!  mychat.org
 (регистрация[token] · федерация[off] · звонки[off] · веб[on] · админ[on] · моб-регистрация[on] · большие-файлы[4G] · движок tuwunel/Rust, без Postgres)

 Регистрация / вход участников
   Веб (рекомендуется): откройте https://mychat.org для регистрации и входа
   Приложение: установите Element X → сервер "mychat.org" → регистрация (код) или вход

 Админ (создан автоматически)
   логин: admin    пароль: xxxxxxxxxxxx
   Веб-панель: https://admin.mychat.org  (вход под учёткой админа выше)

 Ежедневное управление:  sudo tuwunel   (меню)    sudo tuwunel adduser   (добавить участника)
========================================================
```

> 🔑 **Запишите логин/пароль админа и код приглашения!** Всё хранится на сервере в `/opt/tuwunel/CREDENTIALS.txt` (`cat /opt/tuwunel/CREDENTIALS.txt`).
> ⚠️ «Ещё не готово» в конце обычно означает, что облачный фаервол не пропускает 80/443, либо DNS не распространился глобально. Caddy сам повторит выпуск сертификата — переустановка не нужна.

## 👥 Как участники присоединяются (выберите одно)

1. **Админ создаёт аккаунт (максимальный контроль):** `sudo tuwunel adduser` создаёт пользователя и задаёт пароль одной командой; передайте ему «домен + имя пользователя + пароль». Код приглашения не покидает ваших рук.
2. **Раздать код приглашения, саморегистрация через веб:** отправьте код (в `CREDENTIALS.txt`) + `https://ваш-домен`; участники регистрируются сами в браузере.
3. **Раздать код, саморегистрация через Element X:** установите Element X → введите домен → создайте аккаунт → введите код приглашения.

## 🖥️ Графическая панель администратора (Ketesa, рекомендуется)

Если включено (опция 5), установщик развернёт **графическую веб-панель администратора** — всё управляется мышью, без команд.

**Как войти:** откройте `https://admin.ваш-домен` → **войдите напрямую под учёткой админа `admin` + пароль** (отдельного «пароля-шлюза» нет; панель привязана к вашему серверу).

**Что можно делать:** управлять пользователями (создать/деактивировать/сбросить пароль), **выпускать и отзывать коды приглашений**, просматривать комнаты/медиа, каталог комнат, запланированные задачи.

> 📌 Страницы «Жалобы / Пожаловавшиеся пользователи» могут выдавать ошибку или быть пустыми — tuwunel не реализует эту функцию; это **нормально** и почти никогда не понадобится.
> 📌 Панели нужен tuwunel v1.8.1+ (образ `:latest` уже содержит). Если вход в панель даёт ошибку: `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`.
> 📌 Добавить панель на существующий сервер: `sudo tuwunel enable-admin` (сначала добавьте DNS-запись `admin`).

## 🔧 Повседневное управление: меню

После установки **запуск `sudo tuwunel` открывает меню управления** (команды запоминать не нужно):

```bash
sudo tuwunel
```

```
┌──────────────────────────────────────────────┐
│  меню управления tuwunel   ваш-домен          │
└──────────────────────────────────────────────┘
  1) Статус                    6) Обновить образы сервисов
  2) Добавить участника        7) Очистить диск
  3) Изменить конфигурацию     8) Перезапустить сервисы
  4) Вкл./выкл. веб-панель     9) Обновить скрипт + новые функции
  5) Сделать бэкап сейчас     10) Полностью удалить
  p) Усиление приватности / очистка метаданных
  s) Затереть файл учётных данных (антифорензика)
  b) Автоматический шифрованный бэкап по расписанию
  0) Выход
```

Или используйте команды напрямую:

```bash
sudo tuwunel adduser          # добавить участника
sudo tuwunel config           # изменить конфигурацию (Enter = оставить; данные/аккаунты нетронуты)
sudo tuwunel update           # подтянуть свежий скрипт с GitHub и применить новые функции (данные нетронуты)
sudo tuwunel enable-admin     # добавить веб-панель на существующий сервер
sudo tuwunel enable-elementx  # включить саморегистрацию через Element X
sudo tuwunel autobackup       # включить еженедельные автоматические шифрованные бэкапы
sudo tuwunel privacy          # приватность/метаданные: посмотреть, что можно убрать, очистить логи
sudo tuwunel forget-secrets   # антифорензика: затереть пароль/код приглашения в открытом виде на диске
sudo tuwunel uninstall        # удалить
```

> 🔄 **Перенастройка без переустановки** (меню 3): заново спрашивает опции, **Enter = оставить текущее значение**, затем перезапуск. Аккаунты, история и ключи сохраняются.

## 🔒 Приватность / антифорензика (акцент этой редакции)

Система создана для защиты конфиденциальной связи и **по умолчанию минимизирует следы, остающиеся на сервере**:

- ✅ **Содержимое сообщений зашифровано сквозным шифрованием** — ни вы (оператор), ни хостинг-провайдер не могут его прочитать. Это единственная гарантия, не зависящая от того, хорош сервер или плох.
- ✅ **Минимизация метаданных (вкл. по умолчанию):** реальный IP клиента не сохраняется (`ip_source`), **удаления действительно необратимы** (оригинал не хранится 60 дней), присутствие выкл., уровень логов слишком низкий для записи IP, доступ к профилям/каталогу комнат ужесточён.
- ✅ **Антифорензические инструменты:** `sudo tuwunel forget-secrets` затирает пароль/код в открытом виде на диске; `sudo tuwunel autobackup` создаёт бэкапы с шифрованием AES-256.

**Честные пределы (сообщайте их клиентам; не обещайте лишнего):**

- ❌ **Метаданные, которые нельзя удалить:** членство в комнатах, лента событий, факт существования аккаунта, имена комнат / файлов — сервер обязан их хранить, чтобы работать. E2E защищает *содержимое*, **а не метаданные** (кто с кем, когда, имена комнат, имена файлов).
- ❌ **Если диск физически изъят / склонирован:** на обычном VPS данные пишутся на диск в открытом виде, и метаданные извлекаются форензикой. Защита этого слоя требует доверенного провайдера/юрисдикции либо продвинутого полнодискового шифрования / фронтового «щита»-прокси (не автоматически; нужна инженерная работа).
- ❌ **Скрыть IP сервера через Cloudflare практически невозможно** (логи прозрачности сертификатов и субдомены в сером облаке его выдают). Реально скрыть IP можно самостоятельным фронтовым WireGuard-прокси.

> Точная формулировка для клиентов: **«Содержимое сообщений и вложения зашифрованы сквозным шифрованием; оператор не может их прочитать. Однако сервер хранит метаданные связи (кто в какой комнате, когда, имена файлов, имена комнат), которые могут быть извлечены при физическом изъятии сервера.»**

## 💾 Резервные копии (настоятельно рекомендуется)

Потеряете базу данных или ключи — **восстановления не будет**. В этой редакции нет PostgreSQL: бэкап — это просто архив `data/tuwunel` (БД + медиа) + `tuwunel.toml` + `.env`.

**Рекомендуется: включите автоматические шифрованные бэкапы** (еженедельно, AES-256, с ротацией, пропускаются при почти полном диске):

```bash
sudo tuwunel autobackup     # задайте папку / глубину хранения / частоту; ключ будет показан — сохраните его в менеджере паролей
```

**Или сделайте бэкап сейчас** (пункт меню 5; можно задать парольную фразу шифрования).

**Скачать на компьютер** (выполните на своей машине):

```bash
scp root@IP_ВАШЕГО_СЕРВЕРА:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **Сохраните ключ бэкапа.** Без него шифрованный бэкап нельзя открыть никогда. Ключ хранится только на сервере: если сервер пропал, а ключ вы не сохранили, бэкапы бесполезны.
> 💡 Локальная папка `backups/` исчезнет вместе с сервером. Регулярно копируйте файлы `.enc` в другое место или укажите папку бэкапов на смонтированный внешний том / объектное хранилище.

## ❓ Частые вопросы

| Проблема | Причина и решение |
|---|---|
| **Element X на телефоне не подключается / «couldn't connect to this homeserver»** | **Сначала проверьте дату и время телефона** (Настройки → Дата и время → «Автоматически»): сбитые часы валят проверку сертификата; это самая частая причина (особенно на Android). Затем: попробуйте другую сеть, убедитесь, что в браузере телефона открывается `https://ваш-домен/.well-known/matrix/client`, и обновите Element X. |
| **Element X пишет «нужно обновление для поддержки службы аутентификации»** | Речь о регистрации. Проверьте, что мобильная регистрация включена (`sudo tuwunel enable-elementx`) и tuwunel обновлён (`docker compose pull tuwunel`). Либо регистрируйте через веб / `adduser`, а участники пусть **входят по паролю** в Element X. |
| **Большие файлы не загружаются** | Проверьте лимит (`MAX_UPLOAD=10G sudo -E tuwunel config`); при Cloudflare хост **`matrix` должен быть в сером облаке** (лимит 100 МБ оранжевого облака убивает большие файлы). |
| **Панель `admin.домен` не открывается / ошибки входа** | Обычно нет DNS-записи `admin` либо сертификат ещё не готов — добавьте запись и подождите пару минут. При ошибках входа: `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`. |
| Забыт пароль админа | `cat /opt/tuwunel/CREDENTIALS.txt`; либо `sudo tuwunel adduser` для нового админа. |
| Изменить конфигурацию (звонки, регистрация, размер файла, админ) | Без переустановки: `sudo tuwunel config` — Enter оставляет значения, данные/аккаунты нетронуты. |
| Звонки соединяются, но нет звука/картинки | В 99% случаев не открыт порт **7882/UDP** — добавьте его в фаерволе провайдера. |
| Диск заполняется | Пункт меню 7 «Очистить диск»; при активном использовании выделите большой том или объектное хранилище. |
| «Не удаётся расшифровать» старые сообщения в зашифрованной комнате | Нормально: у нового устройства нет исторических ключей. Подтвердите новую сессию со старого устройства. |
| Удаление / переустановка | `sudo tuwunel uninstall` (двойное подтверждение перед удалением); переустановка = удалить и снова запустить команду установки. |

## 📦 Серверные компоненты

После установки работают следующие компоненты, все оркестрируются через Docker, с открытым кодом и доступны для аудита:

```
Caddy (автоматический HTTPS) + tuwunel (Rust, встроенный RocksDB, без PostgreSQL)
  + Element Web (ваш веб-клиент, опционально)
  + Ketesa (графическая панель администратора, опционально)
  + LiveKit + lk-jwt-service (звонки, опционально)
```

Каталог установки `/opt/tuwunel`. Вся логика — в одном скрипте `matrix-tuwunel-installer.sh`, который вы можете проверить сами.

## 🆚 Редакция tuwunel против Synapse

В репозитории есть и редакция Synapse (`matrix-installer.sh`). Кратко:

- **Редакция tuwunel (этот документ):** Rust, без PostgreSQL, **эффективнее, сильнее с большими файлами**, 2 ГБ на ~300 человек. Подходит большинству команд, особенно при активной пересылке больших файлов.
- **Редакция Synapse:** Python + PostgreSQL, самая зрелая экосистема и инструменты администрирования, но требовательнее к ОЗУ и слабее с большими файлами.

**Для новых развёртываний рекомендуется редакция tuwunel.**
