# 프라이빗 Matrix 통신 서버 · 원커맨드 배포 (tuwunel 에디션)

**당신의 서버, 당신의 데이터 — 기밀성과 데이터 주권을 위해 만든 셀프호스팅 팀 메신저.**

**tuwunel** 엔진(Rust, 내장 데이터베이스, **PostgreSQL 불필요**) 기반으로 더 가볍고 안정적이며, **Telegram처럼 대용량 파일 / 사진 / 긴 영상을 전송**할 수 있습니다. RAM 2GB로 중형 팀을 넉넉히 운영합니다. 종단간 암호화, 메타데이터 최소화, 초대 전용 가입이 **기본으로 켜져 있습니다**. 명령 한 줄로 배포 — 시스템 관리자가 아니어도 됩니다. 모든 옵션에 쉬운 설명이 붙어 있습니다.

권장 클라이언트: **Element X**. 전용 **VeilX** 클라이언트를 개발 중입니다(더 깔끔하고 쉬우며 기능이 많고, 오픈소스로 감사 가능. 운영팀은 영국·싱가포르·일본 등에 소재). 고객 자료, 계약서, 내부 논의, 음성·영상 회의 — 모두 당신 서버에만 존재합니다. 오픈 프로토콜 [Matrix](https://matrix.org) 기반. 소스는 공개되어 감사 가능(비상업적 무료).

## ✨ 설치하면 무엇을 얻나요

- 💬 텍스트 채팅과 그룹방 (**종단간 암호화 — 서버와 호스팅 업체도 메시지 내용을 볼 수 없음**)
- 📁 **대용량 파일 / 큰 사진 / 긴 영상 전송** (파일당 기본 상한 **4GB**, 더 크게 설정 가능 — 핵심 기능)
- 📞 1:1 및 그룹 음성 / 영상 통화 (선택)
- 📱 휴대폰에서 셀프 가입: **Element X** 설치 → 도메인 입력 → 가입 및 로그인 (element.io 방문 불필요)
- 🌐 자체 도메인 웹 클라이언트: 브라우저에서 `https://당신의-도메인` 접속으로 가입/로그인, 앱 불필요
- 🖥️ **그래픽 웹 관리 콘솔**(Ketesa): 사용자 관리, 초대 코드 발급/철회, 방·미디어 확인
- 🔒 **기본 프라이버시 강화**: 실제 클라이언트 IP 미저장, 삭제(철회)는 영구 삭제, 접속 상태 비공개, 로그에 IP 미기록
- 👥 초대 전용: 기본적으로 초대 코드가 있는 사람만 가입 가능 — 외부인 진입 불가
- ⚡ **자원 효율**: Rust 단일 프로세스, PostgreSQL 불필요 — RAM 2GB로 약 300명 지원

## 📋 시작 전 준비물 (3가지)

| 무엇 | 요건 | 어디서 |
|---|---|---|
| **클라우드 서버(VPS)** | Ubuntu 22.04 / 24.04(또는 Debian 11+), 공인 IP. **팀 규모에 맞춰 RAM 선택**(아래 참조). **대용량 파일 공유 시 충분한 디스크 확보.** | [Vultr](https://www.vultr.com), [DigitalOcean](https://www.digitalocean.com), [Hetzner](https://www.hetzner.com/cloud), [OVHcloud](https://www.ovhcloud.com) 등 신뢰하는 업체 |
| **도메인** | 어떤 확장자든 가능(.com / .org / .net…) | [Namecheap](https://www.namecheap.com), [Porkbun](https://porkbun.com), [Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) |
| **10분** | 전 과정 복사·붙여넣기, 코딩 불필요 | — |

> 💡 **동시 활성 사용자 수로 RAM 선택**(연합 끄기, 채팅+파일 기준): **1GB** ≈ 수십 명(통화 끄기 권장) · **2GB** ≈ 300명 · **4GB / 2 vCPU** ≈ 500명. tuwunel은 가벼워서 CPU가 병목인 경우는 드뭅니다.
> 💡 **대용량 파일에서는 디스크가 진짜 변수입니다.** 미디어는 서버 로컬 디스크에 저장되며, 종단간 암호화 특성상 중복 제거가 불가능합니다. 헤비 유저는 월 수백 GB~TB가 늘 수 있습니다. 시스템 디스크는 ≥ 50–100GB 권장, 헤비 유저는 대용량 볼륨이나 오브젝트 스토리지를 별도 준비하고 "디스크 정리" 메뉴를 확인하세요.
> 💡 **프라이버시에는 관할권이 중요합니다.** 신뢰할 수 있는 업체와 국가를 고르세요 — 원칙적으로 호스트는 협조를 강제당할 수 있습니다. E2E 암호화는 *내용*을 보호하지만, 메타데이터는 호스트에 남습니다.
> 💡 주문 시 반드시 **Ubuntu 22.04 / 24.04** 이미지를 선택하세요. RAM이 2.5GB 미만이면 스크립트가 자동으로 스왑을 추가합니다.

## 0단계 — 운영체제 준비 (제대로 주문했다면 건너뛰기)

**Ubuntu 22.04 / 24.04**(또는 Debian 11+)만 지원합니다.

- **새 서버**: 주문 시 "OS / Image" 항목에서 **Ubuntu 22.04 x86_64**(또는 24.04) 선택.
- **OS가 틀림**: 업체의 "재설치 / Reinstall / Rebuild"로 Ubuntu 22.04 설치. ⚠️ 이 작업은 **모든 데이터를 삭제**합니다. 기존 서버는 먼저 백업하세요.
- 지금 무엇을 쓰는지 모른다면? 접속(3단계) 후 `cat /etc/os-release` 실행.
- 잘못된 OS에 설치해도 스크립트가 감지해 명확히 알려주며, 아무것도 망가뜨리지 않습니다.

## 1단계 — DNS 레코드 추가 (~2분)

**도메인 등록업체**의 DNS 패널에서 **A 레코드**를 추가합니다.

도메인 `mychat.org`, 서버 IP `1.2.3.4` 가정:

| 유형 | 호스트(이름) | 값 | 용도 |
|---|---|---|---|
| A | `@` (루트 도메인) | `1.2.3.4` | 루트 도메인 + 웹 클라이언트 + 위임 |
| A | `matrix` | `1.2.3.4` | 홈서버 |
| A | `admin` | `1.2.3.4` | 웹 관리 콘솔(관리 끄면 생략) |
| A | `livekit` | `1.2.3.4` | 통화(통화 끄면 생략) |
| A | `matrix-rtc` | `1.2.3.4` | 통화(통화 끄면 생략) |

> 📌 호스트 칸에는 **접두사**(`matrix`)만 입력하고, 전체 도메인은 넣지 마세요.
> 📌 레코드는 1~10분 내 전파됩니다. 아직 반영 전이면 설치 프로그램이 자동으로 기다립니다.
> 📌 채팅+파일만(통화 끄기) 쓸 경우 `@` / `matrix` / `admin` 세 개만 필요.
> ⚠️ **Cloudflare 사용 시**: 이 레코드들은 반드시 **회색 구름(DNS only)** 이어야 하며, **주황 구름(프록시)을 켜면 안 됩니다**. 프록시는 ① 대용량 파일을 100MB로 제한(Free/Pro) ② 인증서 발급 차단 ③ 통화 미디어 차단을 유발합니다. 설치 시 CDN 사용 여부를 묻는데(사용 시 "예" → DNS 검사 완화), `matrix` 호스트는 회색 구름을 유지해야 합니다.

## 2단계 — 방화벽 포트 열기 (~1분)

업체의 "보안 그룹 / 방화벽"에서 허용:

| 포트 | 프로토콜 | 용도 |
|---|---|---|
| 80 | TCP | HTTPS 인증서 발급 |
| 443 | TCP + UDP | 웹과 암호화 트래픽 |
| 7881 | TCP | 통화 예비 채널(통화 끄면 생략) |
| **7882** | **UDP** | **통화 음성/영상(가장 빠뜨리기 쉬움! 없으면 소리·화면 안 나옴)** |

> 📌 업체에 "보안 그룹" 계층이 없으면 건너뛰세요 — 스크립트가 시스템 방화벽을 자동 설정합니다.

---

## 3단계 — 서버 접속 (~1분)

터미널을 엽니다(macOS: "터미널", Windows: "PowerShell"). 아래를 실행(IP 교체)하고 비밀번호를 입력하세요(**비밀번호 입력 시 화면에 아무 문자도 안 보이는 것이 정상**):

```bash
ssh root@서버_IP
```

첫 접속에서 지문 확인이 뜨면 `yes` 입력.

> 📌 기본 사용자가 root가 아니면(예: `ubuntu`) `ssh ubuntu@IP` 사용. 스크립트가 자동으로 권한 상승합니다.

---

## 4단계 — 설치 실행 (~5–10분, 자동)

접속 후 **한 줄 전체를 복사**해 터미널에 붙여넣습니다:

```bash
sudo apt-get update && sudo apt-get install -y wget && wget -O tuwunel.sh https://raw.githubusercontent.com/VeilXofficial/veilx_matrix_ocs/main/matrix-tuwunel-installer.sh && sudo bash tuwunel.sh
```

마법사를 따라가세요. **6개 옵션 + CDN 질문**을 묻고, 각 항목은 화면에 설명이 있으니 잘 모르면 Enter를 누르세요(기본값이 가장 안전한 조합):

| 옵션 | Enter(권장) | 설명 |
|---|---|---|
| 1 가입 방식 | **초대 코드 필요** | 코드가 있는 사람만 가입, 최초 가입자가 관리자. (완전 개방도 있으나 업무용 비권장) |
| 2 연합(Federation) | **끔** | 섬 모드: 외부인이 멤버에게 메시지 불가, 공격면 최소. |
| 3 음성/영상 통화 | **먼저 끔** | DNS 두 개(`livekit`/`matrix-rtc`)와 포트 7881/7882 추가 필요. 먼저 채팅+파일부터 안정화. |
| 4 웹 클라이언트 | **켬** | 멤버가 `https://당신의-도메인`으로 가입/로그인, 앱 불필요. |
| 5 웹 관리 콘솔 | **켬** | `admin.당신의-도메인`에 그래픽 콘솔 Ketesa. |
| 6 최대 파일 크기 | **4G** | 원하는 값(예: 10G) 가능. 클수록 디스크 소모 증가. |
| ＋ Cloudflare/CDN 뒤에 있나요 | **아니오** | CDN 사용 시에만 "예"(DNS 검사 완화). `matrix`는 회색 구름 유지. |

> 🔒 **자동으로 켜짐(묻지 않음, 설치 즉시 적용):** ① 새 방은 **E2E 암호화 강제** ② **메타데이터 최소화**(실제 IP 미저장, 삭제 영구화, 접속 상태 끔, IP 미기록 로깅) ③ **Element X 휴대폰 셀프 가입**(네이티브 OIDC, 가입 시 초대 코드 여전히 필요). 메타데이터 강화를 끄려면: `PRIVACY=0 sudo -E tuwunel config`.
> 🤖 **완전 무인 설치:** `REG_MODE=token ENABLE_CALLS=1 ENABLE_ADMIN=1 sudo -E bash tuwunel.sh mychat.org`

이후 자동 진행: DNS 확인 → Docker 설치 → 시스템 튜닝 → 방화벽 → 설정 생성 → 서비스 시작 → HTTPS 인증서 발급 → **관리자 자동 생성 및 계정/비밀번호 출력**.

성공 화면:

```
========================================================
 🎉 tuwunel 배포 완료!  mychat.org
 (가입[token] · 연합[off] · 통화[off] · 웹[on] · 관리[on] · 모바일가입[on] · 대용량파일[4G] · 엔진 tuwunel/Rust, Postgres 없음)

 멤버 가입 / 로그인
   웹(권장): https://mychat.org 접속해 가입·로그인
   모바일 앱: Element X 설치 → 서버 "mychat.org" → 가입(초대 코드) 또는 로그인

 관리자(자동 생성됨)
   계정: admin    비밀번호: xxxxxxxxxxxx
   웹 관리 콘솔: https://admin.mychat.org  (위 관리자 계정으로 로그인)

 일상 관리:  sudo tuwunel   (메뉴)    sudo tuwunel adduser   (멤버 추가)
========================================================
```

> 🔑 **관리자 계정/비밀번호 + 초대 코드를 반드시 적어 두세요!** 모두 서버의 `/opt/tuwunel/CREDENTIALS.txt`에 있습니다(`cat /opt/tuwunel/CREDENTIALS.txt`).
> ⚠️ 끝에 "아직 준비 안 됨"이 나오면 대개 클라우드 방화벽이 80/443을 허용하지 않았거나 DNS가 전역 전파되지 않은 것입니다. Caddy가 인증서를 자동 재시도하므로 재설치는 불필요합니다.

> 해결: 휴대폰 **설정 → 날짜 및 시간 → "자동 설정" 켜기** 후 재시도.

## 👥 멤버 추가 방법 (택1)

1. **관리자가 계정 생성(가장 통제 가능):** `sudo tuwunel adduser`로 계정 생성 및 비밀번호 설정을 한 번에. "도메인 + 사용자명 + 비밀번호"를 전달. 초대 코드는 유출되지 않음.
2. **초대 코드 배포, 웹 셀프 가입:** 코드(`CREDENTIALS.txt`에 있음) + `https://당신의-도메인`을 전달하면 멤버가 브라우저에서 직접 가입.
3. **초대 코드 배포, Element X 셀프 가입:** Element X 설치 → 도메인 입력 → 계정 생성 → 초대 코드 입력.

## 🖥️ 그래픽 관리 콘솔 (Ketesa, 권장)

설치 시(옵션 5 켬) **그래픽 웹 관리 콘솔**이 자동 배포됩니다. 명령 없이 마우스로 모두 관리.

**접속:** `https://admin.당신의-도메인` → **관리자 계정 `admin` + 비밀번호로 바로 로그인**(별도 게이트 비밀번호 없음. 콘솔은 당신 서버에 고정됨).

**할 수 있는 것:** 사용자 관리(생성/비활성화/비밀번호 재설정), **초대 코드 발급 및 철회**, 방·미디어 확인, 방 디렉터리, 예약 작업.

> 📌 "신고 이벤트 / 신고된 사용자" 페이지는 오류나 빈 화면이 날 수 있습니다 — tuwunel이 해당 기능을 구현하지 않았기 때문이며 **정상**입니다. 거의 쓸 일이 없습니다.
> 📌 콘솔은 tuwunel v1.8.1+ 필요(`:latest` 이미지에 포함). 콘솔 로그인 오류 시 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`.
> 📌 기존 서버에 콘솔 추가: `sudo tuwunel enable-admin`(먼저 `admin` DNS 레코드 추가).

## 🔧 일상 관리: 메뉴

설치 후 **`sudo tuwunel` 실행 = 관리 메뉴 열기**(외울 명령 없음):

```bash
sudo tuwunel
```

```
┌──────────────────────────────────────────────┐
│  tuwunel 관리 메뉴   당신의-도메인             │
└──────────────────────────────────────────────┘
  1) 상태 보기               6) 서비스 이미지 업그레이드
  2) 멤버 추가               7) 디스크 정리
  3) 설정 변경               8) 모든 서비스 재시작
  4) 웹 관리 콘솔 켜기/끄기   9) 스크립트 업데이트 + 신기능 적용
  5) 지금 백업              10) 완전 제거
  p) 프라이버시 강화 / 메타데이터 정리
  s) 평문 자격증명 파일 마스킹(안티포렌식)
  b) 자동 예약 암호화 백업
  0) 종료
```

또는 명령을 직접 사용:

```bash
sudo tuwunel adduser          # 멤버 추가
sudo tuwunel config           # 설정 변경(Enter=유지, 데이터/계정 유지)
sudo tuwunel update           # GitHub에서 최신 스크립트를 받아 신기능 적용(데이터 유지)
sudo tuwunel enable-admin     # 기존 서버에 웹 관리 콘솔 추가
sudo tuwunel enable-elementx  # Element X 휴대폰 셀프 가입 켜기
sudo tuwunel autobackup       # 주간 자동 암호화 백업 켜기
sudo tuwunel privacy          # 프라이버시/메타데이터: 삭제 가능 항목 보기, 로그 정리
sudo tuwunel forget-secrets   # 안티포렌식: 디스크의 평문 비밀번호/초대 코드 마스킹
sudo tuwunel uninstall        # 제거
```

> 🔄 **재설치 없이 재구성**(메뉴 3): 옵션을 다시 묻고 **Enter=현재 값 유지**, 이후 재시작. 계정·기록·키 모두 보존.

## 🔒 프라이버시 / 안티포렌식 (이 에디션의 핵심)

이 시스템은 기밀 통신 보호를 위해 설계되었으며, **기본적으로 서버에 남는 흔적을 최소화**합니다:

- ✅ **메시지 내용은 종단간 암호화** — 운영자(당신)도, 호스팅 업체도 읽을 수 없습니다. 서버의 좋고 나쁨과 무관한 유일한 보장입니다.
- ✅ **메타데이터 최소화(기본 켬):** 실제 클라이언트 IP 미저장(`ip_source`), **삭제(철회)는 진짜 영구 삭제**(원본 60일 보관 안 함), 접속 상태 끔, 로그 레벨을 낮춰 IP 미기록, 프로필/방 디렉터리 노출 축소.
- ✅ **안티포렌식 도구:** `sudo tuwunel forget-secrets`로 디스크의 평문 비밀번호/초대 코드 마스킹; `sudo tuwunel autobackup`은 AES-256으로 백업 암호화.

**정직한 한계(고객에게 그대로 설명하고, 과장하지 마세요):**

- ❌ **제거 불가능한 메타데이터:** 방 멤버십, 이벤트 타임라인, 계정 존재 여부, 방 이름 / 파일 이름 — 서버가 작동하려면 반드시 보관해야 합니다. E2E는 *내용*을 보호할 뿐, **메타데이터(누가 누구와, 언제, 방 이름, 파일 이름)는 보호하지 않습니다**.
- ❌ **디스크가 물리적으로 압수/복제되면:** 일반 VPS에서는 데이터가 평문으로 저장되어 포렌식으로 메타데이터 추출이 가능합니다. 이 계층 방어는 신뢰할 수 있는 업체/관할권, 또는 고급 전체 디스크 암호화 / 전면 "실드" 프록시(자동 아님, 엔지니어링 필요)가 필요합니다.
- ❌ **Cloudflare로 서버 IP를 숨기는 것은 사실상 효과 없음**(인증서 투명성 로그, 회색 구름 서브도메인이 노출). 진짜로 IP를 숨기려면 셀프호스팅 WireGuard 전면 프록시가 필요합니다.

> 고객용 정확한 문구: **"메시지 내용과 첨부는 종단간 암호화되어 운영자도 읽을 수 없습니다. 다만 서버는 통신 메타데이터(누가 어느 방에, 언제, 파일 이름, 방 이름)를 보관하며, 서버가 물리적으로 압수되면 추출될 수 있습니다."**

## 💾 백업 (강력 권장)

데이터베이스나 키를 잃으면 **복구 불가**입니다. 이 에디션은 PostgreSQL이 없으므로, 백업은 `data/tuwunel`(DB + 미디어) + `tuwunel.toml` + `.env`를 묶는 것입니다.

**권장: 자동 암호화 백업 켜기**(주간, AES-256, 자동 로테이션, 디스크가 거의 차면 자동 건너뜀):

```bash
sudo tuwunel autobackup     # 폴더/보관 개수/주기 설정. 키가 표시되니 반드시 비밀번호 관리자에 저장
```

**또는 지금 백업**(메뉴 5, 암호화 암구호 설정 가능).

**PC로 다운로드**(본인 컴퓨터에서 실행):

```bash
scp root@서버_IP:/opt/tuwunel/backups/*.enc ~/Desktop/
```

> 🔑 **백업 키를 반드시 보관하세요.** 키가 없으면 암호화 백업은 영원히 열 수 없습니다. 키는 서버에만 존재하므로, 서버가 사라지고 키도 없으면 백업은 무용지물입니다.
> 💡 로컬 `backups/` 폴더는 서버와 함께 사라집니다. `.enc` 파일을 정기적으로 다른 곳에 복사하거나, 백업 폴더를 마운트된 외부 볼륨 / 오브젝트 스토리지로 지정하세요.

## ❓ 자주 묻는 질문

| 문제 | 원인과 해결 |
|---|---|
| **휴대폰 Element X 연결 안 됨 / "홈서버에 연결할 수 없습니다"** | **먼저 휴대폰 날짜·시간을 확인**(설정→날짜 및 시간→"자동"): 시계가 틀리면 인증서 검증 실패, 가장 흔한 원인(특히 Android). 다음으로: 다른 네트워크 시도, 휴대폰 브라우저에서 `https://당신의-도메인/.well-known/matrix/client` 열리는지 확인, Element X 최신 업데이트. |
| **Element X가 "인증 서비스 지원을 위해 업그레이드 필요"라고 함** | 가입 관련입니다. 모바일 가입이 켜졌는지(`sudo tuwunel enable-elementx`)와 tuwunel 최신 여부(`docker compose pull tuwunel`) 확인. 또는 웹 / `adduser`로 가입시키고 멤버는 Element X에서 **비밀번호 로그인**. |
| **대용량 파일 업로드 안 됨** | 파일 크기 상한 확인(`MAX_UPLOAD=10G sudo -E tuwunel config`); Cloudflare 사용 시 **`matrix` 호스트는 회색 구름 필수**(주황 구름 100MB 상한이 대용량 파일을 막음). |
| **관리 콘솔 admin.도메인 안 열림 / 로그인 오류** | 보통 `admin` DNS 레코드 누락 또는 인증서 미발급. 레코드 추가 후 몇 분 대기. 로그인 오류는 `cd /opt/tuwunel && docker compose pull tuwunel && docker compose up -d`. |
| admin 비밀번호 분실 | `cat /opt/tuwunel/CREDENTIALS.txt`; 또는 `sudo tuwunel adduser`로 새 관리자 생성. |
| 설정 변경(통화, 가입, 파일 크기, 관리) | 재설치 불필요: `sudo tuwunel config` — Enter는 값 유지, 데이터/계정 그대로. |
| 통화는 연결되나 소리·화면 없음 | 99%는 **7882/UDP** 미허용. 업체 방화벽에서 추가. |
| 디스크가 참 | 메뉴 7 "디스크 정리"; 헤비 유저는 대용량 볼륨 또는 오브젝트 스토리지 준비. |
| 암호화 방의 오래된 메시지 "복호화 불가" | 정상: 새 기기에 과거 키가 없음. 기존 기기에서 새 세션을 검증하세요. |
| 제거 / 재설치 | `sudo tuwunel uninstall`(이중 확인 후 삭제); 재설치 = 제거 후 설치 명령 재실행. |

## 📦 서버 측 구성요소

설치 후 다음 구성요소가 실행됩니다. 모두 Docker로 오케스트레이션되며 오픈소스로 감사 가능:

```
Caddy(자동 HTTPS) + tuwunel(Rust, RocksDB 내장, PostgreSQL 없음)
  + Element Web(자체 웹 클라이언트, 선택)
  + Ketesa(그래픽 관리 콘솔, 선택)
  + LiveKit + lk-jwt-service(통화, 선택)
```

설치 경로 `/opt/tuwunel`. 모든 로직은 단일 스크립트 `matrix-tuwunel-installer.sh`에 있으며 직접 감사할 수 있습니다.

## 🆚 tuwunel 에디션 vs Synapse

저장소에는 Synapse 에디션(`matrix-installer.sh`)도 있습니다. 요약:

- **tuwunel 에디션(본 문서):** Rust, PostgreSQL 없음, **더 효율적이고 대용량 파일에 강함**, 2GB로 약 300명. 대부분의 팀, 특히 대용량 파일을 많이 쓰는 경우에 적합.
- **Synapse 에디션:** Python + PostgreSQL, 생태계와 관리 도구가 가장 성숙하지만 RAM을 더 쓰고 대용량 파일에 약함.

**신규 배포는 tuwunel 에디션 권장.**
